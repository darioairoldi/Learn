Azure Container Linux, or ACL, brings together Flatcar's proven, vendor-neutral, and immutable design with the hardened security and trusted supply chain of Azure Linux. The result is a secure, scalable container host, purpose-built for running modern workloads in production. ACL is based on Flatcar's architecture and delivers a minimal, immutable operating system optimized for containers. While it isn't binary compatible with Flatcar, since it's built on Azure Linux rather than Gen2 sources, it provides the same core functionality and operational model that customers rely on today. Where ACL really stands out is in its built-in, out-of-the-box security guarantees. First, immutability is enforced at the kernel level.
Critical system components like the usr directory are locked down to prevent drift and unauthorized changes. Second, ACL uses a minimal, purpose-built package set, reducing the overall attack surface and lowering exposure to CVEs. Third, it includes strong runtime protection through SELinux, enforcing strict least-privilege access controls across the system. On top of that, ACL supports Trusted Launch with a unified kernel image, ensuring the integrity of the system from the very first stage of boot. And finally, ACL benefits from the same secure end-to-end Microsoft supply chain as Azure Linux, using signed RPMs that you can trust. Together, this gives customers a modern, immutable, and security-first foundation for running container workloads on AKS at scale.
