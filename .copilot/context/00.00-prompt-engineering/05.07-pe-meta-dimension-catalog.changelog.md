# Change history — 05.07-pe-meta-dimension-catalog

## v1.5.0 — 2026-06-01

Dimension table now uses canonical `D#-readable-id` form in the `ID` column (e.g. `` `D1-metadata` ``) and the redundant `#` column was dropped. Aligns the table with the canonical-reference-form contract already declared in the file body; the table previously contradicted that contract by holding bare readable suffixes (e.g. `` `metadata` ``). Schema change for table consumers; no semantic dimension change. (Plan: src/docs/90. Issues/202606/20260601.02-dim-readable-ids/02-align-dimids-usecases-pemeta-plan.md follow-up.)

## v1.4.0 — 2026-06-01

Declared `D#-readable-id` as canonical reference form outside this catalog. Rewrote Dimension groups and Dimension-to-handler mapping tables to use the combined form. Broadened boundaries clause to cover BOTH numeric prefix and kebab-case suffix as contract. (Plan: src/docs/90. Issues/202606/20260601.02-dim-readable-ids/01-dimids-rename-plan.md Phase 1.)
