# Change history — 05.07-pe-meta-dimension-catalog

## v2.1.0 — 2026-06-25

Sharpened `D4-tool-alignment` to require the **minimal necessary tool set** (each declared tool injects ~100–500 tokens/step; no over-permissioning; prefer tool-set aliases). Extended `D20-token-chain` to note always-on context is billed every interaction and agent step. Added an **artifact-level vs runtime efficiency scope boundary** note clarifying that `--dim efficiency`/`@meta-optimizer` optimize artifacts, while runtime/user-session habits (interaction-mode sizing, Markdown conversion, within-session cache stability) live in `02.02` guidance and are out of the optimizer's mandate. (Issue: src/docs/90. Issues/202606/20260625.06-github-copilot-token-optimization — plan 03.)

## v2.0.0 — 2026-06-22

**BREAKING (group contract).** Redefined the four primary `--dim` groups as a **disjoint partition** that together cover all 35 dimensions: `strategic` = `D15-vision-alignment` only (was D15+D16+D17+D19); `quality` expanded to 18 dims (added D1,D2,D5,D12,D13,D14,D16,D17,D18,D19,D22 alongside the existing D6–D11,D27) = "is the artifact good / goal alignment"; `efficiency` trimmed to 8 dims (removed D7,D9,D11 — pure quality properties) = "low latency + token efficiency"; `reliability` unchanged (D28–D35). **Removed the `optimize` group** — its apply-delegation to `@meta-optimizer` is now reached via `--dim efficiency --mode apply` (the optimizer agent itself is unchanged). Added a "Which `--dim` group should I pick?" decision guide and a coverage-guarantee note. Sharpened `D14-craftsmanship` (surface craft only; sizing/single-responsibility belongs to `D19-artifact-structure`). Reworked the Dimension-to-handler mapping "Optimize group" row into an "Efficiency apply" row. Convenience groups (`structural`, `freshness`, `adherence`, `model`, `context-full`, `context-health`) retained as overlapping subsets. (Issue: src/docs/90. Issues/202606/20260622.01-pe-meta-update.)

## v1.5.0 — 2026-06-01

Dimension table now uses canonical `D#-readable-id` form in the `ID` column (e.g. `` `D1-metadata` ``) and the redundant `#` column was dropped. Aligns the table with the canonical-reference-form contract already declared in the file body; the table previously contradicted that contract by holding bare readable suffixes (e.g. `` `metadata` ``). Schema change for table consumers; no semantic dimension change. (Plan: src/docs/90. Issues/202606/20260601.02-dim-readable-ids/02-align-dimids-usecases-pemeta-plan.md follow-up.)

## v1.4.0 — 2026-06-01

Declared `D#-readable-id` as canonical reference form outside this catalog. Rewrote Dimension groups and Dimension-to-handler mapping tables to use the combined form. Broadened boundaries clause to cover BOTH numeric prefix and kebab-case suffix as contract. (Plan: src/docs/90. Issues/202606/20260601.02-dim-readable-ids/01-dimids-rename-plan.md Phase 1.)
