# Critical Rules Priority Matrix

**Purpose**: Ranked enforcement checklist for validators. Check CRITICAL rules first, then HIGH, then MEDIUM. Each rule links to its canonical source — don't restate the full rule here, use this as a lookup index.

**Referenced by**: All validator agents (`pe-prompt-validator`, `pe-agent-validator`, `pe-context-validator`, `pe-instruction-validator`, `pe-skill-validator`, `pe-hook-validator`, `pe-prompt-snippet-validator`), `pe-meta-validator`, PE-validation skill

**📖 Rule precedence across layers** (which layer wins when rules conflict): [01.01-context-engineering-principles.md](01.01-context-engineering-principles.md) → "Rule Precedence Across Layers"

## How to use this matrix

### For validators (per-artifact validation)

1. Load the target artifact
2. Run all **CRITICAL** checks (C1–C7) — BLOCK if any fail
3. Run all **HIGH** checks applicable to the artifact type — flag failures
4. Run **MEDIUM** checks — note findings
5. Report with severity counts: `CRITICAL: N, HIGH: N, MEDIUM: N, LOW: N`

### For pe-meta-validator (ecosystem audit)

1. Run CRITICAL checks across all artifacts in scope
2. Calculate health score: `100 - (CRITICAL×25 + HIGH×10 + MEDIUM×3 + LOW×1)`
3. Report findings grouped by severity

### Severity → action mapping

| Severity | Action | Blocks approval |
|---|---|---|
| CRITICAL | MUST fix immediately | Yes |
| HIGH | Fix before production use | No (but flagged) |
| MEDIUM | Fix when convenient | No |
| LOW | Note for future review | No |
