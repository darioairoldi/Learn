# pe-meta Review Dimension Catalog

## Canonical reference form

The canonical reference form for any dimension outside this catalog is `D#-readable-id` (for example `D6-consistency`, `D12-staleness`, `D26-model-routing`). The numeric prefix is preserved for grep-based tooling and contract continuity; the kebab-case suffix is the value from the `ID` column of the dimension table below.

- Combined form (preferred citation): `D6-consistency`
- Bare numeric form (accepted as input, e.g. for `--dim`): `D6`
- Bare readable form: NOT used (always carry the `D#` prefix)
- Ranges: rewrite as explicit lists (`D6-consistency through D11-actionability`) or as the dimension-group name when the row's intent is the group

Historical changelog entries that describe a past state may retain the bare `D#` form.

| ID | What it checks | Deterministic? | Model class | Applicable to |
|---|---|---|---|---|

| `D2-references` | All `📖`, markdown links, and slash-commands resolve | ✅ Yes | Deterministic | All types |
| `D3-token-budget` | Per-artifact token count within type-specific limit | ✅ Yes | Deterministic | All (except hooks) |
| `D4-tool-alignment` | plan=read-only, agent=read+write; tool count 3-7 | ✅ Yes | Deterministic | Agents, prompts |
| `D5-boundaries` | Three-tier completeness (≥5/2/3 exemplary bar) | ⚠️ Partial | Standard | Agents |
| `D6-consistency` | Non-contradiction within file and across loaded dependencies | ❌ No | Reasoning | All types |
| `D7-non-redundancy` | Each rule defined in exactly one canonical location | ⚠️ Partial | Standard | Context, instructions |
| `D8-prioritization` | Conflict precedence explicit in priority matrix | ⚠️ Partial | Standard | Context, instructions |
| `D9-clarity` | Rules unambiguous — LLM agreement test (2-pass) | ❌ No | Reasoning | Context, instructions |
| `D10-completeness` | No gaps where guidance is needed but absent | ❌ No | Reasoning | Context, instructions |
| `D11-actionability` | Rules translatable to boolean checks | ❌ No | Standard | Context, instructions |
| `D12-staleness` | Source references current; timestamps fresh; external sources valid | ⚠️ Partial | Standard + web | All types |
| `D13-source-verification` | Cited sources exist and support the claims made | ❌ No | Reasoning + web | Context, instructions |
| `D14-craftsmanship` | N-1 separation, structure quality, naming conventions | ⚠️ Partial | Standard | All types |
| `D15-vision-alignment` | Artifact complies with applicable vision rationales | ❌ No | Reasoning | All (except hooks, snippets) |
| `D16-adherence` | Artifact implements rules from its dependencies | ❌ No | Reasoning | Agents, prompts |
| `D17-cross-coherence` | Loaded dependencies don't contradict each other; peer mode for context files | ❌ No | Reasoning | Context*, agents, prompts |
| `D18-coverage` | Use cases covered; unhappy paths handled; error recovery defined | ❌ No | Reasoning | Agents, prompts |
| `D19-artifact-structure` | Goal/scope/boundaries correctly sized; single responsibility | ❌ No | Reasoning | All (except hooks, snippets) |
| `D20-token-chain` | Total loading cost for consumer's guidance set within context window | ⚠️ Partial | Standard | Agents, prompts |
| `D21-deterministic-first` | Process phases optimally split between deterministic tools and LLM (deterministic-processing) | ❌ No | Reasoning | Agents, prompts |
| `D22-context-optimization` | Context file SET organized optimally: layer correctness, granularity, orphans | ⚠️ Partial | Reasoning | Context (system-level) |
| `D23-reference-efficiency` | Reference count, load necessity, load placement | ⚠️ Partial | Standard | Agents, prompts |
| `D24-handoff-efficiency` | Handoff contracts have max tokens; payloads use summarization | ⚠️ Partial | Standard | Agents, prompts |
| `D25-processing-efficiency` | Targeted reviews, summarization between phases, model routing per step | ⚠️ Partial | Standard | Prompts |
| `D26-model-routing` | Each process step uses the correct model class for its task type | ❌ No | Standard | Agents, prompts |
| `D27-model-adherence` | Prompting patterns follow target model best practices | ❌ No | Reasoning | Agents, prompts |
| `D28-reproducibility` | Process produces same output for same input across runs (R3 — process reproducibility) | ⚠️ Partial | Reasoning | Agents, prompts |
| `D29-regression-protection` | Loop stability + change-impact gate: edits do not regress prior invariants or oscillate (R1 + R4) | ❌ No | Reasoning | All types (except hooks, snippets) |
| `D30-metadata-guard` | Metadata invariants are enforced (R5 frontmatter + R6 dimension/group contracts) | ⚠️ Partial | Standard | All types |
| `D31-multipass-validation-invariant` | Validation re-runs converge (no flapping between PASS/FAIL on stable input) (R2) | ❌ No | Reasoning | Agents, prompts |
| `D32-rollback-readiness` | Mutating commands declare and exercise rollback paths (R8) | ❌ No | Reasoning | Agents, prompts |
| `D33-boundary-actionability` | Boundaries are red-team-resistant: each "NEVER/ALWAYS" rule is checkable and bypass-resistant (R7) | ❌ No | Reasoning | Agents, prompts |
| `D34-autonomy-calibration` | Autonomy level matches risk class; `--mode apply` defaults match documented mutation scope (R9) | ❌ No | Reasoning | Agents, prompts |
| `D35-portability-boundary` | Portability claims match actual coupling; cross-repo/cross-model boundaries declared (R11) | ❌ No | Reasoning | Agents, prompts, context |

*`D17-cross-coherence` for context files = peer-review mode (checks against peer context files loaded by same consumers)

## Dimension groups (shortcuts)

| Group | Dimensions | Use when |
|---|---|---|
| `--dim full` | All 35 (`D1-metadata` through `D35-portability-boundary`) | Complete audit (default if no `--dim` specified) |
| `--dim structural` | `D1-metadata` through `D5-boundaries`, `D14-craftsmanship` | Quick structural pass — cheapest useful review |
| `--dim quality` | `D6-consistency`, `D7-non-redundancy`, `D8-prioritization`, `D9-clarity`, `D10-completeness`, `D11-actionability`, `D27-model-adherence` | Guidance quality (all 6 prerequisite properties) + model adherence |
| `--dim strategic` | `D15-vision-alignment`, `D16-adherence`, `D17-cross-coherence`, `D19-artifact-structure` | Vision alignment + adherence + structure review |
| `--dim freshness` | `D12-staleness`, `D13-source-verification` | Staleness and source verification |
| `--dim efficiency` | `D3-token-budget`, `D4-tool-alignment`, `D7-non-redundancy`, `D9-clarity`, `D11-actionability`, `D20-token-chain`, `D21-deterministic-first`, `D23-reference-efficiency`, `D24-handoff-efficiency`, `D25-processing-efficiency`, `D26-model-routing` | Full efficiency assessment |
| `--dim adherence` | `D5-boundaries`, `D6-consistency`, `D16-adherence`, `D18-coverage` | Boundaries + consistency + adherence + coverage. Canonical name as of v13. |
| `--dim context-full` | `D1-metadata` through `D3-token-budget`, `D6-consistency` through `D15-vision-alignment`, `D17-cross-coherence`, `D19-artifact-structure`, `D22-context-optimization` | Full context file review (16 applicable dims) |
| `--dim context-health` | `D6-consistency` through `D12-staleness`, `D22-context-optimization` | Quick context quality + staleness + organization |
| `--dim model` | `D26-model-routing`, `D27-model-adherence` | Model routing + model-specific adherence |
| `--dim optimize` | `D3-token-budget`, `D7-non-redundancy`, `D9-clarity`, `D11-actionability`, `D20-token-chain`, `D21-deterministic-first`, `D23-reference-efficiency`, `D24-handoff-efficiency`, `D25-processing-efficiency`, `D26-model-routing` | Performance optimization; efficiency-lens review; delegates apply to `@meta-optimizer` |
| `--dim reliability` | `D28-reproducibility` through `D35-portability-boundary` | System-reliability validation: reproducibility, regression protection, metadata guards, validation invariants, rollback readiness, boundary actionability, autonomy calibration, portability boundaries (goal-trio reliability pole) |

## Dimension-to-handler mapping

| Type | Handler | Why |
|---|---|---|
| Deterministic (`D1-metadata` through `D4-tool-alignment`) | pe-meta-validator with deterministic tools | Parse YAML, count lines, resolve paths |
| Structural (`D5-boundaries`, `D14-craftsmanship`) | pe-meta-validator with standard model | Pattern matching against known structures |
| Semantic (`D6-consistency` through `D11-actionability`, `D16-adherence` through `D19-artifact-structure`) | pe-meta-validator with reasoning model | Requires understanding meaning |
| External (`D12-staleness`, `D13-source-verification`) | pe-meta-researcher with `fetch_webpage` | Requires accessing external sources |
| Cross-artifact (`D17-cross-coherence`, `D20-token-chain`) | pe-meta-validator in dependency-aware mode | Requires reading multiple files |
| Efficiency (`D21-deterministic-first`, `D23-reference-efficiency` through `D25-processing-efficiency`) | pe-meta-validator with standard model | Analyzing reference patterns and process phases |
| Model (`D26-model-routing`, `D27-model-adherence`) | pe-meta-validator with reasoning model | Comparing patterns against model-specific guidance |
| System-level (`D22-context-optimization`) | pe-meta-validator in system mode | Requires reading ALL context files |
| Reliability (`D28-reproducibility` through `D35-portability-boundary`) | pe-meta-validator with reasoning model + deterministic helpers | `D30-metadata-guard` uses deterministic parsers; `D28-reproducibility`/`D29-regression-protection`/`D31-multipass-validation-invariant` through `D35-portability-boundary` require reasoning over process invariants |
| Optimize group | pe-meta-validator (review) → @meta-optimizer (apply) | Review uses standard efficiency dims; apply delegates to optimizer agent |
