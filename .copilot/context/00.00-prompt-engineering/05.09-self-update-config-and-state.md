# Self-update config and state substrate

## Purpose

Specify the two artifacts the portable self-update engine needs to make autonomy decisions on evidence rather than guesswork: a declarative **config** (what to watch, how bold to be, where to keep state) and a persisted **state namespace** (what happened, so confidence can grow). This closes gap **G5** (config + state substrate absent). It is a **contract only** — the build is parked; nothing reads these files until that work is unparked.

## Referenced by

- **PE vision** — the autonomy gradient and `progressive-learning` principles (the consumers of the thresholds + history)
- **`00.06-folder-metadata-inheritance.md`** — `domain_profile.authoritative_sources` is the per-domain source of truth this config references by domain id

## `pe-self-update.config.json`

A single portable file at the engine install root. Spec-now / build-later; consumers MUST preserve unknown keys (forward-compat via `config_version`).

| Key | Type | Purpose |
|---|---|---|
| `config_version` | int | Contract version for forward-compat (REQUIRED). |
| `engine_id` | scalar | Stable id; selects the `.copilot/state/<engine_id>/` namespace. |
| `monitored_sources` | `{ <domain_id>: [{ url, version_scheme, alert_keywords }] }` | Per-domain Type-B source registry. Each `<domain_id>` SHOULD resolve to a domain whose `domain_profile.authoritative_sources` is the authoring source of truth; config MAY add or override at runtime. |
| `autonomy_thresholds` | `{ <change_class>: { min_confidence, action } }` | Maps assessed change classes to an action (`auto` / `propose` / `escalate`). Absent ⇒ most conservative (`escalate`). |
| `state_root` | path | Where to write the state namespace (default `.copilot/state/`). Resolved relative to install root. |

## `.copilot/state/<engine_id>/` layout

Per-engine, namespaced, append-mostly. Never shared between engine instances.

| Path | Contents | Consumed by |
|---|---|---|
| `outcome-log.jsonl` | One line per self-update attempt: change class, decision, confidence, eval result, human override. | `progressive-learning` (confidence inputs) |
| `rename-log.jsonl` | Every artifact rename/move the engine performed, with old→new + reason. | `metadata-guarded-changes`, link integrity |
| `snapshots/<artifact>@<hash>/` | Pre-change snapshot enabling rollback + before/after diffing. | rollback, eval baselining |

## How config + state drive autonomy

1. **Detect** uses `monitored_sources` (per the declaring artifact's domain) to fire Type-B staleness.
2. **Assess** classifies the change; `autonomy_thresholds` + the `outcome-log` success rate for that class yield `auto` / `propose` / `escalate`.
3. **Execute** snapshots first, applies, records the outcome (and any human override) so the next decision is better-grounded — the `progressive-learning` loop.

Config absent ⇒ no monitored sources and the conservative default tier (`escalate`): the engine degrades to fully-supervised, never silently autonomous.

## Status

**Spec-now / build-later.** This file defines the contract; the config reader, state writer, and rollback tooling are parked (see the plan's § Park lot). Until then the substrate is inert and the engine runs fully supervised.
