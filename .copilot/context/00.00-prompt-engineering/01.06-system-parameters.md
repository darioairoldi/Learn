# PE System Parameters

**Purpose**: Single source of truth for all quantitative thresholds used across PE artifacts. When a context file, agent, or prompt needs a specific limit or budget, it MUST reference this file instead of hardcoding the value.

**Referenced by**: All context files, instruction files, agents, prompts, skills, validators

---

## Token Budgets

| Parameter | Value | Applies to | Rationale |
|---|---|---|---|
| Context file max | 2,500 tokens (~375 lines) | All files in `.copilot/context/` | Keeps on-demand loading cost manageable |
| Instruction file max | 1,500 tokens (~220 lines) | All files in `.github/instructions/` | Auto-injected — must stay small |
| Agent file max | 2,500 tokens (~375 lines) | All files in `.github/agents/` | Loaded on @mention — similar to context files |
| Simple prompt max | 1,500 tokens (~220 lines) | Single-agent prompts, create-update, review prompts | Injected per execution |
| Orchestrator prompt max | 2,500 tokens (~375 lines) | Multi-agent design prompts with handoffs + summarization | Complex coordination needs space |
| Meta-prompt max | 3,000 tokens (~450 lines) | `meta-prompt-engineering-*` prompts | System-wide orchestration |
| SKILL.md description max | 100 tokens | `description:` field in SKILL.md | Level 1 discovery — always visible to AI |
| SKILL.md body max | 1,500 tokens | SKILL.md body content | Level 2 instructions — loaded on activation |

## Context Rot Thresholds

| Parameter | Value | Source |
|---|---|---|
| Accuracy degradation threshold | 32,000 tokens → accuracy drops from 88% to 30% | Liu et al. (2023) "Lost in the Middle" |
| Single request baseline | ~1,500 tokens (1,000 input + 500 output) | Measured baseline |
| 5-phase workflow unoptimized | ~14,300 tokens (9.5× single request) | Token multiplication analysis |

## Agent Boundaries

| Parameter | Value | Applies to |
|---|---|---|
| Always Do minimums | ≥ 3 items | All agent files |
| Ask First minimums | ≥ 1 item | All agent files |
| Never Do minimums | ≥ 2 items | All agent files |
| Tool count range | 3–7 tools | All agent and prompt files |
| Tool count warning | > 7 tools causes tool clash risk | Agents with many tools |

## Validation Rules

| Parameter | Value | Applies to |
|---|---|---|
| Validation cache duration | 7 days | Bottom metadata `last_run` timestamp |
| Cache skip condition | < 7 days AND content unchanged | All validation prompts |
| Deprecation grace period | 30 days | Deprecated artifacts before removal |
| Use case challenge scenarios | 3–5 per artifact | Creation and review validators |
| Role validation scenarios | 3–7 per agent | Agent researchers |

## Phase Budgets (Orchestrated Workflows)

| Phase | Target max | Action if exceeded |
|---|---|---|
| Research | 3,000 tokens | Summarize to 1,000 tokens before handoff |
| Architecture/Design | 2,000 tokens | Split into multiple specs |
| Build | 4,000 tokens | Use file-based output, reference by path |
| Validation | 1,500 tokens | Compress to issues-only report |

## Information Flow Thresholds

| Parameter | Value | Applies to |
|---|---|---|
| Full context (`send: true`) recommended for | 2–3 phase workflows | Simple orchestrations |
| Progressive summarization recommended for | 3–5 phase workflows | Medium orchestrations |
| File-based isolation recommended for | 5+ phase workflows | Long orchestrations |

## Content Length Guidelines

| Context file type | Token budget | Word count | Line count |
|---|---|---|---|
| Core principles | 800–1,200 tokens | ~600–900 words | ~120–180 lines |
| Pattern libraries | 1,500–2,500 tokens | ~1,100–1,875 words | ~220–375 lines |
| Workflow documentation | 1,000–2,000 tokens | ~750–1,500 words | ~150–300 lines |
| Terminology/Glossary | 500–1,000 tokens | ~375–750 words | ~75–150 lines |
| Maximum recommended | 2,500 tokens | ~1,875 words | ~375 lines |

> **Note:** Orchestrator prompts (2,500 tokens) and meta-prompts (3,000 tokens) have higher budgets — see Token Budgets table above.

## Conversion Reference

| Metric | Approximate conversion |
|---|---|
| Tokens → Words | Divide by 1.33 |
| Tokens → Lines | Divide by 6–8 (avg 6.67) |
| Tokens → Characters | Multiply by 4 |
| Words → Tokens | Multiply by 1.33 |

## Combined Context Budget

When a prompt references agents and instructions, consider cumulative impact:

| Scenario | Components | Total | Status |
|---|---|---|---|
| **Comfortable** | Prompt (800) + Agent (1,000) + 3 instructions (900) + question (50) + code (2,000) | ~4,750 | ✅ |
| **Warning** | Prompt (1,800) + Agent (1,500) + 6 instructions (1,800) + question (100) + code (5,000) | ~10,200 | ⚠️ |

| Budget range | Status | Action |
|---|---|---|
| < 10,000 tokens | Optimal | Proceed |
| 10,000–15,000 tokens | Warning | Consider simplification |
| > 15,000 tokens | Critical | Refactor required |
