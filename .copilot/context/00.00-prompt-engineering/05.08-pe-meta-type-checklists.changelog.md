# Change history — 05.08-pe-meta-type-checklists

## v1.5.0 — 2026-06-25

Scoped the Prompt-files "Test scenarios (≥3)" check to **orchestrator-class** prompts only (user-invoked entry points carrying `--option` flags); **per-type sub-prompts** (e.g. `pe-meta-agent-design`, `pe-meta-prompt-review`) are now N/A — they rely on the invoking orchestrator's scenario suite plus their own `argument-hint` example. Resolves the cohort-wide F2 finding from the `/pe-meta-review --mode plan` of `pe-meta-agent-design`/`pe-meta-agent-review` (plan 01-pe-meta-review-agent-design-review): per-type prompts no longer false-flag for missing embedded test scenarios.

## v1.4.0 — 2026-06-12

Added a metadata-field-placement note after § How to use — `version:`/`last_updated:` checklist rows are evaluated against the bottom `{type}_metadata` block (not top frontmatter) per 00.03-metadata-contracts, with the vision/use-case top-frontmatter exception called out. Closes plan 01-pe-artifacts-changelog-fix item 6 reader-layer clarification.

## v1.2.0 — 2026-06-11

Back-ported the sub-check enumeration from the Agent-files checklist to the seven remaining artifact-type checklists (issue 20260607.02 doc 02, plan 02.03 I-B). Promoted the Agent-specific sub-check-granularity blockquote to a single global note after § How to use (single-source — removed the duplicate Agent-table blockquote). Added the `Deterministic?` column to the Context, Prompt, Instruction, Skill, Template, Hook, and Snippet tables (column order now `Check | Standard bar | Exemplary bar | Deterministic? | Dimension` across all eight tables). Added composite sub-check rows where a non-agent type had a hidden sub-check: Prompt files gained Scope-enforcement-step (D5-boundaries), Claim/implementation-match (D6-consistency), No-re-hosted-contract (D14/D23), and Section-proportionality (D19); Context files gained No-re-hosted-rule (D14/D23). Closes the false-clean class of defect on non-agent artifact types.

## v1.1.0 — 2026-06-11

Added the `Deterministic?` column and the H14-alignment, operationalize-not-restate, claim/implementation-match, no-re-hosted-contract, and section-proportionality sub-check rows to the Agent-files table (issue 20260607.02 doc 02, plan 02.02 I1). NOTE: this bump was applied to the frontmatter `version:` but this bottom-block entry was added retroactively in v1.2.0 to reconcile a desync.

## v1.0.5 — 2026-06-07

Added a `D30-metadata-guard` check to the Agent-files table — bottom metadata key MUST be `agent_metadata:` (canonical per in-scope majority; `article_metadata:` is the article convention). Closes the 2026-06-07 follow-up I4 (metadata-key canonical-source discipline).

## v1.0.4 — 2026-06-07

Scoped the instruction-file 'Severity index present' check — added a scope note clarifying it applies only to instruction files governing PE artifacts validated by the global matrix, and is N/A for document-governance instruction files (use-case/vision). Closes pe-meta-update D1 by clarification.

## v1.0.3 — 2026-06-01

Rewrote `Dimension` column from bare `D#` to canonical `D#-readable-id` form per Plan 01-dimids-rename-plan Phase 3.
