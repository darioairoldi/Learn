# PE Practical Effectiveness Log

**Purpose**: Captures user-reported outcomes from PE workflow executions. Provides external evidence for meta-researcher to validate and refine PE rules against practical results — not just structural compliance.

**Referenced by**: `meta-researcher` agent, `pe-meta-scheduled-review` prompt, `meta-validator` agent

---

## How to Use This Log

### Adding entries (after running a PE workflow)

After using any PE workflow (`/pe-gra-prompt-design`, `/pe-gra-agent-review`, `/pe-gra-context-information-create-update`, etc.), add a brief entry:

```markdown
### YYYY-MM-DD — [workflow name]

- **Goal**: [what you were trying to accomplish]
- **Outcome**: [worked well / partial / failed]
- **Friction points**: [what was slow, confusing, or required workarounds — or "None"]
- **Artifacts affected**: [files created/modified]
```

### Reading entries (during meta-reviews)

`meta-researcher` MUST read this log during Step 1 to factor real user outcomes into improvement recommendations. Patterns to look for:

- Workflows with repeated "partial" or "failed" outcomes → rules or agent behavior needs refinement
- Recurring friction points → structural improvement opportunity
- Workflows with consistent "worked well" → rules are effective, don't change without cause
