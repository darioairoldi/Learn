# Appendix: Unified Agent Architecture (v1.107+)

> **Parent article:** [How to structure content for GitHub Copilot agent files](./04.00-how_to_structure_content_for_copilot_agent_files.md)
>
> **Related concept:** [Chat modes, Agent HQ, and execution contexts](../03-concepts/01.08-chat_modes_agent_hq_and_execution_contexts.md)

VS Code 1.107 introduced a paradigm shift in how agents execute, providing a <mark>unified interface for managing agent sessions</mark> across multiple execution contexts.

## Agent HQ Interface

<mark>**Agent HQ**</mark> is the centralized interface for managing all agent sessions:

| Feature | Functionality |
|---------|---------------|
| **Recent Sessions List** | Displays local, background, and cloud sessions with unified status |
| **Read/Unread Markers** | Blue dots indicate unread completed sessions |
| **Filtering & Search** | Filter by session type, search across session history |
| **Archive Capability** | Move completed sessions to archive for cleaner workspace |
| **Expand to Sidebar** | Full sidebar view for detailed session management |

## Execution Context Details

### Local Agents
- Execute in the current VS Code session
- Interactive—user can observe and intervene in real-time
- No isolation—changes affect workspace directly
- Best for: Quick tasks, iterative refinement, learning agent behavior

### Background Agents
- Execute autonomously without blocking VS Code
- Use <mark>**Git work trees**</mark> for complete filesystem isolation
- Sessions continue even when VS Code loses focus
- Changes reviewed and merged after completion
- Best for: Long-running tasks, parallel work on multiple features

**Work Tree Workflow:**
1. Create background agent session (dropdown option to use work tree)
2. Agent works in isolated work tree—no conflicts with local changes
3. Work tree appears in Source Control view
4. Review changes after completion
5. Use "Apply" action to integrate changes into main workspace

### Cloud Agents
- Execute on GitHub infrastructure
- Automatically create branch and pull request
- Fully asynchronous—can close VS Code entirely
- Best for: Large changes, team collaboration, code review workflows

## Delegation Flow

The unified architecture enables seamless delegation between execution contexts:

```
Planning Mode → Local Agent → Background Agent → Cloud Agent
      ↓              ↓              ↓               ↓
   Creates      Implements     Isolates via      Creates
    Plan         Locally      Work Trees     Branch + PR
```

**"Continue In" options** available from:
- Chat view "Continue in" button
- Planning mode delegation dropdown
- Untitled prompt file button

## Chat Customizations Editor (Preview)

VS Code provides a centralized <mark>**Chat Customizations editor**</mark> for managing all customization files from a single UI. Open it via the gear icon in the Chat view or **Command Palette → Chat: Open Chat Customizations**.

| Tab | What you manage | Actions available |
|-----|----------------|-------------------|
| **Instructions** | `.instructions.md` files | Create, edit, view `applyTo` patterns |
| **Prompts** | `.prompt.md` files | Create, edit, test |
| **Agents** | `.agent.md` files | Create, edit, configure tools |
| **Skills** | `SKILL.md` folders | Create, browse resources |
| **MCP Servers** | `mcp.json` configurations | Browse marketplace, install, manage |
| **Hooks** | `.github/hooks/*.json` | Create, configure lifecycle events |
| **Agent Plugins** | Pre-packaged bundles | Browse marketplace, install |

The editor includes an embedded code editor with syntax highlighting and validation. You can configure customizations per agent type: local agents, Copilot CLI, and the Claude agent.

## Agent Design by Execution Context

When designing custom agents, tailor your YAML frontmatter and instructions for the intended execution context:

### Local agent design

```yaml
---
name: interactive-reviewer
description: "Code review with interactive feedback loops"
tools: ['codebase', 'problems', 'usages', 'changes']
model: claude-sonnet-4
---
# Interactive Code Reviewer
Review code changes and discuss findings with the developer.
Ask clarifying questions when intent is ambiguous.
```

**Design principles for local agents:**
- Include `ask_questions`-style interaction in instructions (the model can ask the user)
- Use read-only tools unless editing is the primary task
- Keep instructions conversational — the user is present and can redirect

### Background agent design

```yaml
---
name: refactoring-agent
description: "Autonomous refactoring with comprehensive testing"
tools: ['codebase', 'editor', 'filesystem', 'terminal', 'problems']
model: claude-opus-4.6
---
# Autonomous Refactoring Agent
Complete the refactoring task independently. Run tests after every change.
Write a summary of all changes to `.copilot/scratch/refactor-summary.md`.
Do NOT ask the user questions — make reasonable decisions autonomously.
```

**Design principles for background agents:**
- Include clear success criteria so the agent knows when it's done
- Instruct the agent to write results to files (the user reviews after completion)
- Use a capable model (the agent can't ask for help)
- Add `terminal` tool so the agent can run tests and validate

### Cloud agent design (AGENTS.md)

```markdown
<!-- AGENTS.md (repository root) -->
# Agent Instructions

## Code Style
- Follow the project's ESLint configuration
- Use TypeScript strict mode
- Write tests for all new functions

## PR Guidelines
- Create focused, single-purpose commits
- Include a clear PR description
- Reference the issue number in commit messages
```

**Design principles for cloud agents:**
- Use `AGENTS.md` (not `.agent.md`) — cloud agents use a different file format
- Set `target: github-copilot` in `.agent.md` files intended for cloud delegation
- Focus on success criteria and coding standards (the agent works asynchronously)
- Include PR conventions — the output is a pull request, not a chat response

## Agent Plugins (Preview)

<mark>**Agent plugins**</mark> are pre-packaged bundles of customizations you can install from a marketplace. A single plugin can provide:

- Slash commands (prompt files)
- Agent skills
- Custom agents
- Hooks
- MCP servers

Discover and install plugins via the Chat Customizations editor → Agent Plugins tab, or browse the marketplace directly.
- Untitled prompt file button

## Claude Skills Support

VS Code 1.107 added <mark>**Claude Skills**</mark> support, enabling agents to leverage the growing Claude skills ecosystem for extended capabilities. This allows custom agents to access specialized tools and behaviors defined in Claude skill files.

## Impact on Agent Design

When designing custom agents, consider the execution context:

| Agent Type | Recommended Context | Rationale |
|------------|---------------------|----------|
| **Planning agents** | Local | Interactive refinement of plans |
| **Implementation agents** | Background or Cloud | Long-running, benefits from isolation |
| **Review agents** | Local | Interactive feedback needed |
| **Documentation agents** | Background | Can run while developer works |

**Best Practices:**
- Use `target: github-copilot` for agents intended for cloud execution
- Design agents to work autonomously for background execution
- Include clear success criteria so background/cloud agents know when complete
- Consider using handoffs to transition from planning (local) to implementation (background/cloud)
