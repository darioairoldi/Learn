> **Status: done (2026-06-11).** Both items applied and validated clean. **I-A** — `pe-meta-evidence-coverage.md` gained a § Unified recorded-state enum (one enum for the `(artifact × dimension)` state: graded verdict `verified`/`pass-weak`/`partial`/`fail` + the `never` un-exercised sentinel) with the legacy `pass|fail|partial|never` → graded mapping; the Outcome table was reconciled; `pe-meta-update.prompt.md` (v2.8.0→v2.9.0, also backfilling the missed v2.8.0 bottom-block changelog entry) reworded the inherited-PASS phrasings and added a unified-enum pointer to the `subcheck-coverage` field def. **I-B** — `05.08` (v1.1.0→v1.2.0) promoted the granularity note to a single global note, added the `Deterministic?` column to all seven non-agent tables, and added composite sub-check rows (Prompt: D5 scope-step, D6 claim-match, D14/D23 no-re-host, D19 proportionality; Context: D14/D23 no-re-host).

## 🎯 Goal

Remove the two residual incoherences the 02.02 implementation left behind:

1. **Two enums for one axis.** The per-PU coverage `status` (`pass | fail | partial | never`) and the per-dimension graded verdict (`verified | pass-weak | partial | fail`) describe the **same** `(artifact × dimension)` recorded state, yet are spelled differently across artifacts. The landed vision already declares the recorded state IS the graded verdict (`processing-state-is-multidimensional`, line 229) — the downstream artifacts must say so too.
2. **Sub-check enumeration only on agents.** 02.02 added the `Deterministic?` column, the granularity note, and the composite sub-check rows to the **Agent-files** table only. The other seven artifact-type checklists in `05.08` still expose a dimension as a single row, so the same false-clean defect (a composite dimension passing on its easiest sub-check) can recur on a prompt, context, or instruction file.

**Verbatim trigger:** *"proceed with [folding the coverage status enum into the graded-verdict enum] [back-porting sub-check enumeration to non-agent artifact types]"* (2026-06-11).

## 🧭 Design anchors

- **Recorded grain stays per-dimension** (the C1 reversal, upheld by 02.02). Sub-checks are evaluated and consolidated into the dimension verdict; they are **not** a third persisted axis. The fold does not add an axis — it makes one axis use one enum.
- **`never` survives the fold.** `never` is not a quality grade — it is the **un-exercised sentinel** (a PU with no `evidence_ref`, or an inherited PASS deliberately demoted on a re-run). The fold keeps `never` as the orthogonal "no verdict yet" state and replaces the *exercised* states (`pass`/`fail`/`partial`) with the graded verdict (`verified`/`pass-weak`/`partial`/`fail`).
- **Single source of truth.** The grade rubric stays canonical in [pe-meta-evidence-coverage.md](../../../../../../.github/prompt-snippets/pe-meta-evidence-coverage.md); every other artifact references it. The backfill repeats the *structure* (column + rows) per artifact type but never re-hosts the rubric.
