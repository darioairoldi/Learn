## 🎯 Goal

Make a `/pe-meta-update` full pass **unable to report a dimension clean while its harder sub-checks were never executed**, by realizing the graded-verdict model from [overview.md](overview.md) Part 2 in the live pe-meta artifacts.

**Verbatim trigger:** *"please proceed with ## 🅿️ Park lot changes for the vision implementation"* (2026-06-11).

## 🧭 Design anchor — recorded grain stays per-dimension

This plan implements the model **as corrected by the C1 reversal** in the landed vision, NOT the literal Part-2 text:

- The **recorded** processing unit stays `(artifact × dimension)`. A dimension's sub-checks are **evaluated and consolidated** into its graded verdict — they are not persisted as a third PU axis.
- `subcheck-coverage=<evaluated>/<declared>` is a **per-dimension completeness ratio** that feeds the verdict (a ratio < 1 ⇒ `partial`). It does NOT redefine `pu-evidence` away from per-dimension granularity; both markers coexist.
- This matches the vision principles `processing-state-is-multidimensional` (per-dimension recorded state), `evidence-based` (graded verdict consolidates sub-checks), and `dimension-rule-self-containment` (the dimension enumerates every targeting rule).

### Phase 2 — Graded verdict + marker in the evidence contract (I2) (✅ done)

5. In [pe-meta-evidence-coverage.md](../../../../../../.github/prompt-snippets/pe-meta-evidence-coverage.md), keep the PU definition at `(artifact × applicable-dimension)` and add a new subsection **"Sub-check completeness and the graded dimension verdict"** that states:
   - a dimension's `evidence_ref` set MUST cover every sub-check declared for its artifact type in `05.08` (ties to `dimension-rule-self-containment`);
   - `subcheck-coverage=<evaluated>/<declared>` is emitted **per dimension**; `<evaluated> < <declared>` ⇒ the dimension verdict is `partial`.
   - **Acceptance:** the subsection states explicitly that this does NOT redefine `pu-evidence` (still per-dimension) — `subcheck-coverage` is an additional per-dimension ratio.
6. Add the **grade rubric** table to the same file: `verified` (all sub-checks evaluated + pass, property sub-checks backed by property anchors), `pass-weak` (all evaluated + pass but ≥1 property sub-check backed only by a presence anchor), `partial` (ratio < 1 — UNVERIFIED, blocks a clean verdict), `fail` (≥1 sub-check failed). State that `partial` is computed deterministically from the ratio (no LLM, no sampling).
7. Extend **Layer A** with the deterministic **presence-vs-property anchor classification**: a sub-check tagged as a property test whose anchor cites only a heading/count (presence) is flagged, feeding the `pass-weak` grade with zero LLM cost. Reference the existing hook `pe-check-evidence-anchors.ps1` as the evaluation home (extend its checks; do not move the rule out of guidance).
   - **Acceptance:** all four grades are defined; `partial` and `fail` are stated to block a clean score; the presence-vs-property check carries an `Evaluation:` tag consistent with the existing delegation-annotation convention.
