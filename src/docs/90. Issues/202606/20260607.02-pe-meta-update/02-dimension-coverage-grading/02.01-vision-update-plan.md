# Vision update plan — sub-check coverage granularity and graded dimension verdicts

> **Status note.** This plan amends a vision document. Per the vision boundary *"MUST NOT be modified by autonomous processes — vision changes are human-only"*, the proposed wording was authored in `status: draft` for human review. **Applied 2026-06-11 under explicit author direction** ("please proceed with the vision update plan") — V1/V2/V3 landed in `20260531.01-vision.md` (bumped 15.4.0 → 15.5.0, changelog v15.5). Explicit human direction satisfies the human-only-vision boundary (supervised refinement, not autonomous drift). Status: `done`.

## 🎯 Goal

**Verbatim trigger (original):** *"please understand why the full analysis from /pe-meta-update was not able to detect so relevant issues into pe-meta-validator.agent.md agent and how we could change the /pe-meta-update flow (and the vision, if needed) to ensure it will fully cover its goal."*

The root cause established in [overview.md](overview.md): coverage is measured per *dimension*, but a dimension is a *checklist*. "Covered" currently means *touched with one real quote*, so a dimension goes green on its easiest sub-property while harder sub-checks (e.g. the H14 boundary↔YAML alignment rule) are never executed — and every existing guard passes honestly. Closing this requires three vision-level changes; everything else is downstream implementation.

| # | Item | Scope tag | Principle impact | Downstream landing |
|---|---|---|---|---|
| V1 | Extend principle `processing-state-is-multidimensional` so the **processing axis** is per-artifact, per-dimension, **per-sub-check** — a dimension verdict MUST NOT be collapsed to a single per-dimension scalar | `[in-scope: original]` | `preserves: evidence-based, deterministic-where-possible`; `touches: processing-state-is-multidimensional (P1 justification: the principle already forbids collapsing the processing axis to a per-source scalar; this closes the symmetric gap one level down — collapsing a dimension's checklist to one per-dimension scalar is the same failure class.)` | `landing: 06.00-idea/self-updating-prompt-engineering/20260531.01-vision.md` |
| V2 | Refine principle `evidence-based` to introduce a **graded dimension verdict** (`verified` / `pass-weak` / `partial` / `fail`) and the **presence-vs-property evidence distinction**, where `partial` (not all sub-checks evaluated) deterministically BLOCKS a clean verdict | `[in-scope: original]` | `preserves: deterministic-where-possible, single-source-of-truth`; `touches: evidence-based (P1 justification: refines "evidence" to separate presence-evidence from property-evidence and makes unevaluated sub-checks visible as `partial` instead of silently passing.)` | `landing: 06.00-idea/self-updating-prompt-engineering/20260531.01-vision.md` |
| V3 | Add a new P1 principle **`dimension-rule-self-containment`**: a dimension's sub-check set MUST enumerate every quality rule that targets that dimension; no rule may be reachable only from a different file | `[in-scope: original]` | `preserves: single-source-of-truth, runtime-grounding, deterministic-where-possible`; `touches: none existing (additive new P1 principle)` | `landing: 06.00-idea/self-updating-prompt-engineering/20260531.01-vision.md` |

**No P0 principle is touched.** V1 and V2 touch P1 principles (justifications inline above). V3 is additive. None alters the eight-parameter canonical surface (the new `subcheck-coverage` marker is a log marker, not a CLI parameter — `minimal-canonical-surface` (P0) is preserved). No P0 consent line is therefore required; the human-only-vision boundary is satisfied by this plan remaining `draft` for review.

## 🧭 Why these three, and why only these

| Finding behind it | Vision change | Why it must live in the vision (not just implementation) |
|---|---|---|
| Coverage unit is one level too coarse | V1 | The "two-axis, never a scalar" rule is already a *principle*; the sub-check granularity is the same principle, so it belongs at the same level — otherwise the gate has no authority to demand it |
| A run passed breadth + depth + shallow-sweep yet was shallow at the sub-check level; presence evidence ≠ property evidence | V2 | The grade rubric changes what "evidenced" *means*; `evidence-based` is the principle that defines evidence, so the refinement must amend it |
| The H14 alignment rule exists in `00.03` but is never reached from the `D5` definition | V3 | This is a structural invariant about how dimensions relate to rules; without a principle, "wire every targeting rule into the dimension" is just a one-off edit that the next dimension will re-violate |

## 🔧 Proposed vision edits (for review) (✅ applied 2026-06-11 — vision v15.5.0)

These are the concrete text changes for the human to approve. Exact wording is a proposal, not applied.

### V1 — `processing-state-is-multidimensional` (✅ applied)

Append to the statement (the per-source clause stays):

> "…and the processing axis is itself recorded per-artifact, per-dimension, **per-sub-check** — a dimension's verdict is the conjunction of its sub-check states and is never collapsed to a single per-dimension scalar."

### V2 — `evidence-based` (✅ applied)

Append a clause:

> "A dimension's coverage carries a graded verdict — `verified` (all sub-checks evaluated and passed, property sub-checks backed by property evidence), `pass-weak` (all passed but a property sub-check is backed only by presence evidence), `partial` (not all sub-checks evaluated — UNVERIFIED, blocks a clean verdict), `fail` (≥1 sub-check failed). `partial` is computed deterministically from the evaluated/declared sub-check ratio."

### V3 — new P1 principle (✅ applied — `relies_on` corrected to `[instruction-minimization, deterministic-processing, role-declaration]`)

```yaml
- id: dimension-rule-self-containment
  priority: P1
  statement: "A dimension's sub-check set enumerates every quality rule that targets that dimension; a rule that names a dimension is reachable from that dimension's definition, never only from a different file."
  rationale: "A quality rule housed in a file the dimension never loads is never executed — the H14 boundary-alignment rule sat in 00.03 while the D5 definition in 05.07/05.08 was a bare count, so a full pass discharged D5 on the count and never reached H14."
  relies_on: [single-source-of-truth, deterministic-processing, role-declaration]
```

> **`single-source-of-truth` reconciliation.** V3 does NOT duplicate H14 into the dimension. The rule stays canonical in `00.03`; the dimension's sub-check **references** it. This is the same reference-not-restate posture `single-source-of-truth` already requires.

## 🅿️ Park lot

Implementation of the model is **out of scope for this vision plan** and is quarantined here. Each item spawns or feeds the sibling implementation plan once the vision changes are approved.

> **Recorded-grain reconciliation (2026-06-11).** The applied vision (C1 correction) keeps the *recorded* processing unit at `(artifact × dimension)` — a dimension's sub-checks are **evaluated and consolidated into its graded verdict**, not persisted as a third PU axis. The original Part-2 model text ("PU becomes the sub-check") was walked back to per-dimension recorded state because dimension-level state is more robust and the sub-check consolidation is a *verdict computation* (`partial` from the evaluated/declared ratio), not a separate stored axis. The implementation items below are reconciled to this grain: sub-checks are enumerated and evaluated, but `subcheck-coverage` reports per-dimension completeness — it does not redefine the persisted PU.

- **Decompose `05.07` / `05.08` into per-sub-check rows** (D5 → H14 alignment + mode-count + no-re-host rows; add a `Deterministic?` flag per sub-check) → [02.02-coverage-model-implementation.plan.md](02.02-coverage-model-implementation.plan.md)
- **Add the graded-verdict model to `pe-meta-evidence-coverage.md`** — keep PU = `(artifact × dimension)`; add `subcheck-coverage=<evaluated>/<declared>` **per dimension** (a completeness ratio feeding the verdict, NOT a PU redefinition); add the presence-vs-property anchor classification to Layer A; add the `verified`/`pass-weak`/`partial`/`fail` grade rubric where `partial` (ratio < 1) deterministically blocks a clean verdict → [02.02-coverage-model-implementation.plan.md](02.02-coverage-model-implementation.plan.md)
- **Upgrade Phase 7d Coverage Audit** in `pe-meta-update.prompt.md` to RE-EXECUTE sub-checks against the live artifact (adversarial) instead of re-reading the writer's quotes; add a Phase 8 linter rule that `partial` blocks a clean health score → [02.02-coverage-model-implementation.plan.md](02.02-coverage-model-implementation.plan.md)
- **Wire `runtime-grounding` enforcement check into the agent type checklist** (the H14 row was applied to `pe-meta-validator.agent.md` v2.4.0 as F2, but the generic agent checklist still lacks the sub-check) → [02.02-coverage-model-implementation.plan.md](02.02-coverage-model-implementation.plan.md)
