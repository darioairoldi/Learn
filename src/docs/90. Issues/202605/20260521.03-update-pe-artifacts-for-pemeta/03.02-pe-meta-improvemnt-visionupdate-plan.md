### 5. Eliminate `--mode` from scheduled-review (âś… decided)

**Problem:** `pe-meta-scheduled-review` currently uses `--mode individual|dep-aware-full|dep-aware-shallow|guidance-first` to select rotation mode. The new `--mode plan|apply` is execution-mode. These are different semantics sharing the same flag name.

**Resolution: Option D â€” eliminate `--mode` from scheduled-review entirely.**

The scheduled-review's `--mode` values map cleanly to existing option classes:

| Current `--mode` value | Replacement | Rationale |
|---|---|---|
| `individual` | `--deps none` | Per-artifact checks, no dep traversal |
| `dep-aware-full` | `--deps full` | Full dependency chain validation |
| `dep-aware-shallow` | `--deps direct` | First-level dependency checks |
| `guidance-first` | *(internal rotation behavior)* | Auto-delegates to `/pe-meta-adherence` every 4th run |

**Key design decisions:**
1. **`--deps` gains applicability in scheduled-review** â€” passed through to the delegated validator.
2. **`guidance-first` becomes internal rotation logic**, not a user-facing flag. The scheduler auto-delegates to `/pe-meta-adherence` every Nth run. Users who want ad-hoc adherence checking invoke `/pe-meta-adherence` directly.
3. **Auto-rotation updates from**: `individual â†’ dep-aware-full â†’ dep-aware-shallow â†’ guidance-first â†’ repeat` **to**: `--deps none â†’ --deps full â†’ --deps direct â†’ adherence-delegation â†’ repeat`.
4. **User override**: `pe-meta-scheduled-review --deps full` forces full dep-chain review regardless of rotation position.
5. **No `--mode` conflict remains** â€” `--mode plan|apply` is exclusively for update/release-diff commands.

**Composition examples:**
```
/pe-meta-scheduled-review                           â†’ auto (rotation + auto-dim)
/pe-meta-scheduled-review --dim freshness           â†’ freshness dims, auto-rotate deps
/pe-meta-scheduled-review --deps full               â†’ force full dep-chain, auto-select dims
/pe-meta-scheduled-review --dim quality --deps full â†’ quality dims + full dep-chain
```

**Acceptance criteria:**
- Vision documents `--mode` as exclusively `plan|apply` for update/release-diff families. (âś… done)
- Scheduled-review's review methodology controlled by `--deps` (pass-through to validator). (âś… done)
- `guidance-first` documented as internal rotation behavior, not user-facing option. (âś… done)
- Migration path: `--mode individual` â†’ `--deps none`, `--mode dep-aware-full` â†’ `--deps full`, `--mode dep-aware-shallow` â†’ `--deps direct`. (âś… done)
