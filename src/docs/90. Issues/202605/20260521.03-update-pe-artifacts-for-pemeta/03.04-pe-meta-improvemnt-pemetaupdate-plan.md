# PE-meta normalization: implementation update plan

## 🎯 objective (✅ done)

Rewrite the PE-meta implementation artifacts (prompt files, dimension catalog, option applicability matrix) to conform to the normalized option model — making the actual system behavior match the vision's defined taxonomy.

## 📋 prerequisites

- **Workstream 1 (vision) MUST be complete:** supplies command families, option taxonomy, preset alias definitions, applicability principles.
- **Workstream 2 (use cases) MUST be complete:** supplies validated invocation examples that serve as regression test cases.

## 🔎 current state (✅ done)

### Option applicability matrix (`pe-meta-option-applicability-matrix.md`)

Current state (OUTDATED):
- Shows `--scope` restricted to orchestrators only
- Shows `--mode` restricted to `scheduled-review` (rotation semantic, not execution-mode semantic)
- Uses old skip flags: `--skip-research`, `--no-external`, `--skip-stale-check`
- Uses old dependency flags: `--with-deps`, `--with-deps-shallow`
- 5 capability classes (review, review-orchestrate, create/design, release-diff, scheduled-review)

Needed state:
- `--scope` universal (all command families)
- `--mode plan|apply` for update/release-diff families
- Consolidated `--skip <stage>[,<stage>...]` (no more individual skip flags)
- Consolidated `--deps none|direct|full|<N>` (no more individual depth flags)
- Capability classes aligned with command families

### Dimension catalog (`05.07-pe-meta-dimension-catalog.md`)

Current groups: `full`, `structural`, `quality`, `strategic`, `freshness`, `efficiency`, `robustness`, `context-full`, `context-health`, `model`

Missing: **`optimize`** group (D3, D7, D9, D11, D20-D26)

### Update pipeline prompt (`pe-meta-update.prompt.md`)

Line ~350: "restrict to checks 3-5" — needs update to "all 6 checks with efficiency lens when `--dim optimize`"

Mode table (lines 72-78): Shows old mode structure — needs rewrite for `plan|apply` semantic.

Scope & Flags (lines 79-104): Shows old `--skip-*` and `--with-deps*` syntax — needs rewrite.

### Prompt argument hints

Multiple prompts have `argumentHint` values showing deprecated option syntax. All need normalization.

## ⚙️ deliverables (✅ done)

### 1. Rewrite option applicability matrix (✅ done)

**Target file:** `.github/prompts/00.09-pe-meta/pe-meta-option-applicability-matrix.md`

**New structure:**

```markdown
## Canonical option classes

| Class | Option | Values | Default |
|---|---|---|---|
| Dimension | `--dim` | `<group>` or `D<N>` | `full` |
| Dependency | `--deps` | `none\|direct\|full\|<N>` | `none` |
| Target scope | `--scope` | `<type>\|<path>\|all` | `all` |
| Execution mode | `--mode` | `plan\|apply` | `apply` |
| Pipeline control | `--skip` | `research,external,structure,consistency,content` | (none) |

## Applicability matrix

| Command | Family | --dim | --deps | --scope | --mode | --skip |
|---|---|---|---|---|---|---|
| pe-meta-review | Review | ✅ | ✅ | ✅ (filter deps) | ❌ | ❌ |
| pe-meta-{type}-review | Review | ✅ | ✅ | ✅ (filter dep-types) | ❌ | ❌ |
| pe-meta-adherence | Guidance | ❌ | ✅ | ✅ (filter deps) | ❌ | ❌ |
| pe-meta-update | Update | ✅ | ❌ | ✅ (filter artifact-types) | ✅ | ✅ |
| pe-meta-scheduled-review | Scheduled | ✅ | ✅ (pass-through to validator) | ✅ (filter artifact-types) | ❌ | ✅ |
| pe-meta-release-monitor | Release-diff | ✅ | ❌ | ✅ (filter artifact-types) | ✅ | ✅ |
| pe-meta-design | Creation | ❌ | ❌ | ❌ | ❌ | ❌ |
| pe-meta-create-update | Creation | ❌ | ❌ | ❌ | ❌ | ❌ |

## Rejection behavior

When an unsupported option is passed:
1. Stop execution
2. Emit: `⛔ Option --{name} is not supported by {command} (family: {family}). Reason: {command} does not implement {capability}. Try: {suggested_command} which supports --{name}.`
3. Do NOT silently ignore unsupported options
```

**Acceptance criteria:**
- Every PE-meta command has a row in the matrix. (✅ done)
- Every cell is ✅ or ❌ (no "partial" support). (✅ done)
- Rejection message template defined and testable. (✅ done)
- Scope semantics column clarified per command (what scope means for that command). (✅ done)

### 5. Normalize argument parsing instructions (✅ done)

**Add or update the "Parse options" step in each prompt:**

```markdown
## Option parsing

Parse the following options from the invocation:

| Option | Type | Default | Validation |
|---|---|---|---|
| `--dim` | Group name or D-number | `full` | Must be a recognized group or dimension |
| `--deps` | `none\|direct\|full\|<N>` | `none` | Integer N must be ≥1 |
| `--scope` | Type name or path | `all` | Type must be valid artifact type |
| `--mode` | `plan\|apply` | `apply` | Exactly one of the two values |
| `--skip` | Comma-separated stages | (none) | Each stage must be: research, external, structure, consistency, content |

If an unsupported option is detected, reject per the applicability matrix rejection template.
```

**Only prompts that support the option include it in their parsing table.** (Don't add `--mode` parsing to review commands.)

**Acceptance criteria:**
- Every prompt file that accepts options has an explicit parsing section. (✅ done)
- Parsing tables are subset-consistent (only accepted options listed). (✅ done)
- Rejection behavior references the applicability matrix. (✅ done)

```markdown
After expansion, proceed with the resolved flags. User-supplied flags override expanded defaults (e.g., `healthcheck --dim freshness` = `--mode plan --skip research --dim freshness`).
```

### 8. Update pe-meta-scheduled-review to use `--deps` instead of `--mode` (✅ done)

**Decision (from workstream 1 deliverable 5): Option D — eliminate `--mode` entirely, use `--deps` pass-through.**

**Changes required in `pe-meta-scheduled-review.prompt.md`:**

1. **Remove `--mode` from argument-hint YAML:**
   - Old: `"--mode individual|dep-aware-full|dep-aware-shallow|guidance-first"`
   - New: `"--deps none|direct|full"`

2. **Rewrite Phase 2 review mode rotation section:**
   - Old: "Determine review mode (from `--mode` flag, or auto-rotate from review log)"
   - New: "Determine dependency depth (from `--deps` flag, or auto-rotate from review log)"
   - Rotation cycle: `--deps none → --deps full → --deps direct → adherence-delegation → repeat`
   - When `--deps` is user-specified: override rotation position, pass value to validator
   - Every 4th run: delegate to `/pe-meta-adherence` instead of validator (internal behavior)

3. **Update YAML `scope.covers` section:**
   - Old: "Review mode rotation (individual, dep-aware-full, dep-aware-shallow, guidance-first)"
   - New: "Dependency depth rotation (none, full, direct) with periodic adherence delegation"

4. **Update argument-hint:**
   - Old: `'Optional: "--scope ...", "--dim ...", "--mode individual|dep-aware-full|dep-aware-shallow|guidance-first"'`
   - New: `'Optional: "--scope context|agents|prompts|instructions|skills|hooks|snippets|templates", "--dim <group|D#|full>", "--deps none|direct|full"'`

5. **Update review log rotation tracking:**
   - Log entries record `--deps` value used instead of mode name
   - Adherence-delegation runs logged as "adherence-delegation (auto)"

6. **Migration guidance (in prompt comments):**
   - `--mode individual` → `--deps none`
   - `--mode dep-aware-full` → `--deps full`
   - `--mode dep-aware-shallow` → `--deps direct`
   - `--mode guidance-first` → removed (internal rotation behavior)

**Acceptance criteria:**
- `--mode` does not appear anywhere in `pe-meta-scheduled-review.prompt.md`. (✅ done)
- `--deps` accepted and passed through to validator delegation. (✅ done)
- Auto-rotation logic preserved using `--deps` values in review log. (✅ done)
- `guidance-first` behavior preserved as internal 4th-run delegation. (✅ done)
- No breaking change for users (old `--mode` values rejected with migration guidance). (✅ done)
