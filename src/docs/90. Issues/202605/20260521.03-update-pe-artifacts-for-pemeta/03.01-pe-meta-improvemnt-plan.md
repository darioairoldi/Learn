# PE-meta command contract normalization plan

## 🎯 objective (✅ done)

Make PE-meta command options consistent, relevant, and easy to understand by defining one canonical option model, then aligning vision, use cases, and prompt implementations to that model.

## 🔎 current state review summary (✅ done)

### command families and current option surfaces (✅ done)

1. Core review family (`pe-meta-review`, `pe-meta-{type}-review`) supports `--dim`, `--with-deps`, `--with-deps-shallow`.
2. Design and create-update family (`pe-meta-design`, `pe-meta-create-update`, type-specific variants) supports `--dim` only.
3. Scheduled review (`pe-meta-scheduled-review`) supports `--scope`, `--deps` (pass-through to validator, replaces old `--mode` rotation), and `--dim`.
4. Update orchestrator (`pe-meta-update`) supports `--scope`, pipeline mode (`fullcheck|healthcheck|performancecheck`), `--skip-*`, `--plan`, `--no-external`, `--no-research`, `--incremental`, and `--dim`.
5. Release monitor (`pe-meta-release-monitor`) supports `--plan` and `--dim`, then delegates to update semantics.
6. Adherence (`pe-meta-adherence`) is guidance-first canonical and intentionally accepts only guidance target input (unsupported flags rejected).

### consistency gaps to resolve (✅ done)

1. `--mode` conflict resolved: scheduled-review's old `--mode individual|dep-aware-full|dep-aware-shallow|guidance-first` eliminated; replaced by `--deps none|direct|full` pass-through + internal adherence-delegation rotation. `--mode plan|apply` remains exclusively for update/release-diff.
2. `--scope` appears in orchestration commands, but scope grammars are not fully unified across prompts and examples.
3. `--plan` semantics are consistent (preview/no apply), but availability and wording differ between update and release monitor.
4. `--no-external` and `--no-research` are conceptually close but operationally distinct, which is not always explicit in command help.
5. `--skip-*` is update-only, but this capability boundary is not consistently explained as phase-graph ownership.
6. `--incremental` is update-only and currently appears as a performance knob rather than a first-class execution strategy.

## 📋 canonical option model proposal (✅ done)

### option classes (✅ done)

1. Global semantic options: `--dim` (includes new `optimize` group for efficiency/redundancy/budget checks). (✅ done) — Implemented in dimension catalog and all prompts.
2. Review depth options: `--deps none|direct|full|<N>` (single flag replacing `--with-deps` and `--with-deps-shallow`; backward-compatible aliases retained). (✅ done) — Implemented in review prompts and applicability matrix.
3. Scope options: `--scope` (universal; in orchestrators selects which artifact types to iterate; in type-specific commands selects which dependency types to focus on during traversal). (✅ done) — Universal across all prompts with context-dependent semantics.
4. Execution mode options: `--mode plan|apply` (replaces positional pipeline modes; `--plan` retained as alias for `--mode plan`; `--incremental` remains a separate performance knob). (✅ done) — Implemented in pe-meta-update with preset decomposition.
5. Pipeline and resource control: `--skip <stage>[,<stage>...]` (unified flag replacing both `--skip-*` phase flags and `--no-external`/`--no-research` source flags; in orchestrators skips named pipeline phases or cross-cutting resources; in type-specific commands only `research` and `external` stages are meaningful). (✅ done) — Implemented in pe-meta-update with backward-compatible aliases.

### naming and behavior normalization (✅ done)

1. Replace `--with-deps` and `--with-deps-shallow` with unified `--deps` flag across review commands:
   - `--deps` (no value) = bounded recursive traversal, default depth 5 (default behavior when flag is present).
   - `--deps none` = no dependency traversal (explicit opt-out).
   - `--deps direct` = first-level dependencies only (replaces `--with-deps-shallow`).
   - `--deps full` = same as bare `--deps`; explicit synonym for clarity.
   - `--deps <N>` = explicit numeric depth for fine-grained control.
   - Retain `--with-deps` → `--deps full` and `--with-deps-shallow` → `--deps direct` as backward-compatible aliases. (✅ done) — Aliases documented in applicability matrix and pe-meta-update.
2. Make `--scope` universal across all commands with context-dependent semantics:
   - Values: `all` (default), `context`, `instructions`, `agents`, `prompts`, `skills`, `hooks`, `snippets`, `templates`, or a specific file path.
   - In orchestrators (`pe-meta-update`, `pe-meta-scheduled-review`): selects which artifact types to iterate in the batch.
   - In type-specific commands (`pe-meta-review`, `pe-meta-{type}-review`, `pe-meta-design`, `pe-meta-create-update`): selects which dependency types to focus on during `--deps` traversal.
   - Example: `/pe-meta-prompt-review 'aw-series-review.prompt.md' --scope context --deps full` = review the prompt, follow full dependency chain but focus on context dependencies.
   - Example: `/pe-meta-update --scope context --dim health --mode plan` = iterate all context files, check health dims, report only.
   - When omitted: defaults to `all` (no filtering; review all dependency types or iterate all artifact types).
   - `--deps` controls recursion depth; `--scope` controls which files to focus on. They compose orthogonally. (✅ done) — Scope semantics documented in all prompts and applicability matrix.
3. Eliminate positional pipeline modes (`fullcheck|healthcheck|performancecheck`) and decompose into composable flags:
   - `healthcheck` → `--mode plan --skip research` with inherent R-only execution (Research substeps only in audit phases; no Build/Validate substeps; fast diagnosis with health score).
   - `performancecheck` → `--mode apply --dim optimize --skip research,structure,consistency` (all Phase 4R checks evaluated through an efficiency lens; delegates apply to `@meta-optimizer`).
   - `fullcheck` → `--mode apply` (default when no flags specified; all phases, all substeps, per-type builders).
   - Retain `healthcheck`, `performancecheck`, `fullcheck` as documented preset aliases for discoverability.
   - Preset alias behaviors not expressible by flags alone (R-only execution, delegation target) are inherent properties of the preset, documented in the alias table. (✅ done) — Preset aliases with inherent behaviors documented in pe-meta-update Modes section.
4. Introduce `--mode plan|apply`:
   - `--mode plan` = full R-B-V analysis (Research + Build + Validate in audit phases) but stop before apply; presents designed changes with diffs for review (replaces `--plan` flag).
   - `--mode apply` = full R-B-V + apply changes (default).
   - Note: `healthcheck` preset implies R-only execution (no Build/Validate) as an inherent optimization — this is NOT the same as `--mode plan`. Plan designs solutions; healthcheck only diagnoses.
   - Retain `--plan` as backward-compatible alias for `--mode plan`. (✅ done) — --mode plan|apply with plan/healthcheck distinction implemented in pe-meta-update.
5. Add `--dim optimize` group to the dimension catalog:
   - Includes dimensions: D3 (token-budget), D7 (non-redundancy), D9 (clarity), D11 (actionability), D20 (token-chain), D21 (deterministic-first), D23 (reference-efficiency), D24 (handoff-efficiency), D25 (processing-efficiency), D26 (model-routing).
   - When active, Phase 4R evaluates ALL 6 content checks through an efficiency lens ("does this finding represent wasted tokens or reduced signal-per-token?") rather than restricting to checks 3-5. Findings without token/efficiency impact are downgraded to informational.
   - Absorbs the dimension-filtering role previously hidden inside `performancecheck` mode.
   - Triggers delegation to `@meta-optimizer` instead of per-type builders during apply phase. (✅ done) — optimize group added to dimension catalog (D3,D7,D9,D11,D20-D26); performancecheck delegates to @meta-optimizer.
6. Consolidate `--skip-*`, `--no-external`, and `--no-research` into unified `--skip <stage>[,<stage>...]`:
   - Named stages map to pipeline phases and cross-cutting resources:
     - `research` = skip Phase 1 entirely (replaces `--skip-source` and `--no-research`).
     - `external` = skip internet/URL fetching in all phases, keep local artifacts (replaces `--no-external`).
     - `structure` = skip Phase 2 (structure audit; replaces `--skip-structure`).
     - `consistency` = skip Phase 3 (consistency audit; replaces `--skip-consistency`).
     - `content` = skip Phase 4 (content audit; replaces `--skip-content`).
   - Multiple stages are comma-separated: `--skip research,structure` or repeated: `--skip research --skip structure`.
   - In orchestrators (`pe-meta-update`, `pe-meta-scheduled-review`, `pe-meta-release-monitor`): all stages valid.
   - In type-specific commands: only `research` and `external` are meaningful (no multi-phase pipeline).
   - Backward-compatible aliases:
     - `--no-external` → `--skip external`
     - `--no-research` → `--skip research`
     - `--skip-source` → `--skip research`
     - `--skip-structure` → `--skip structure`
     - `--skip-consistency` → `--skip consistency`
     - `--skip-content` → `--skip content`
   - `--incremental` remains a separate performance knob (manifest-based scope filtering, orthogonal to stage skipping). (✅ done) — Unified --skip with backward-compatible aliases implemented in pe-meta-update; --incremental kept orthogonal.
7. Require every prompt to include a "supported options" table with purpose, defaults, valid combinations, and rejected combinations. (✅ done) — All 24 type-specific prompts + 4 orchestrators have consistent "Phase ordering and option behavior" sections (2026-05-24).
