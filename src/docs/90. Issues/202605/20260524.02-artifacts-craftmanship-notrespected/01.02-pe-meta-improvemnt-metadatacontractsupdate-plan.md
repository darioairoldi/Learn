## ⚙️ Deliverables (✅ done)

### Change 1: Add "Runtime grounding protocol" section (✅ done)

**Location:** After "Metadata-guarded change protocol" section

**Proposed content:**

```markdown
## Runtime grounding protocol

Every artifact's body MUST bridge its declarative YAML metadata to imperative runtime behavior. This ensures the executing model enforces its own constraints — not just that validators check them after the fact.

### What runtime grounding means

| YAML field | Runtime enforcement |
|---|---|
| `goal:` | Model verifies each request contributes to the declared goal before proceeding |
| `scope.covers:` | Model confirms the request falls within covered topics |
| `scope.excludes:` | Model rejects requests matching excluded topics with redirection |
| `boundaries:` | Model enforces all MUST/NEVER constraints throughout execution |

### Per-type grounding patterns

| Artifact type | Grounding mechanism | Notes |
|---|---|---|
| **Agent** | Three-tier boundaries section (Always Do / Ask First / Never Do) already acts as runtime grounding when it reflects YAML boundaries | Ensure three-tier content aligns with YAML `boundaries:` |
| **Prompt** | Add scope enforcement step in workflow: "Verify request falls within declared scope before proceeding" | Lightweight — one process step, not a full section |
| **Context** | N/A — context files are reference material, not executed | No grounding needed |
| **Instruction** | N/A — instructions are injected into agent/prompt context, not independently executed | No grounding needed |
| **Skill** | N/A — skills are discovery metadata + instructions, grounding handled by the consuming agent | No grounding needed |
| **Template** | N/A — templates are output scaffolds, not executed | No grounding needed |

### Grounding alignment rule

For agents and prompts: body-level enforcement content MUST be consistent with YAML metadata. If `boundaries:` says "MUST verify handoff targets exist" then the body workflow MUST include a step that performs handoff target verification.

**Violation indicator:** A boundary exists in YAML but has no corresponding enforcement in the body workflow.
```

### Change 2: Add runtime grounding validation rules (✅ done)

**Location:** Validation rules table

**Proposed additions:**

```markdown
| Rule | Severity | Check |
|---|---|---|
| Agent three-tier boundaries align with YAML `boundaries:` | **HIGH** | Each YAML boundary has a corresponding Always Do / Never Do entry |
| Prompt workflow references scope enforcement | **MEDIUM** | Process steps include scope/boundary verification |
| No body-level constraint contradicts YAML metadata | **HIGH** | Body instructions don't override or weaken YAML boundaries |
```
