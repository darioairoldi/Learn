# PE-meta improvement: pe-common instructions update plan — runtime grounding rule

## 🎯 Objective

Update instruction files to:

1. Add a **runtime grounding rule** to `pe-common.instructions.md` (severity: HIGH)
2. Update `pe-prompts.instructions.md` `[C6]` to include universal metadata fields (`goal`, `scope`, `boundaries`, `version`)
3. Ensure `pe-agents.instructions.md` three-tier boundaries pattern explicitly references YAML alignment

## 📋 Prerequisites

- Workstream 2 (metadata contracts update) completed — defines what runtime grounding means per artifact type

## 🔎 Current state analysis

### `pe-common.instructions.md` (v1.6.0)

**Severity index currently:**
- C1: Tool alignment
- C3: Token budget
- C5: No circular deps
- C7: Never modify top YAML

**Missing:** No rule about runtime grounding — body enforcing YAML metadata.

### `pe-prompts.instructions.md` (v1.4.1)

**[C6] currently:** "YAML frontmatter: name, description, agent mode, tools required"

**Missing:** `goal`, `scope`, `boundaries`, `rationales` (SHOULD), `version` — all mandated by metadata-contracts.

### `pe-agents.instructions.md`

Already has three-tier boundaries pattern (Always Do / Ask First / Never Do). But no rule says "three-tier MUST align with YAML `boundaries:`."

## ⚙️ Deliverables (✅ done)

### Change 1: Add runtime grounding rule to `pe-common.instructions.md` (✅ done)

**Location:** Severity Index → HIGH section

**Proposed addition:**

```markdown
- **[H14]** Runtime grounding: body MUST enforce declared YAML metadata at execution time.
  - Agents: three-tier boundaries MUST reflect all YAML `boundaries:` entries
  - Prompts: workflow MUST include scope verification step before proceeding
  - YAML `scope.excludes` MUST have corresponding rejection behavior in body
```

### Change 2: Update Quality Checklist (✅ done)

**Location:** Quality Checklist section

**Proposed addition:**

```markdown
- [ ] Body enforces declared YAML boundaries at runtime (H14)
```

### Change 3: Update `pe-prompts.instructions.md` [C6] (✅ done)

**Current:**
```markdown
- **[C6]** YAML frontmatter: name, description, agent mode, tools required
```

**Proposed:**
```markdown
- **[C6]** YAML frontmatter: name, description, agent mode, tools, goal, scope (covers + excludes), boundaries, version required. rationales SHOULD be included.
```

### Change 4: Add runtime grounding to prompt required sections (✅ done)

**Location:** `pe-prompts.instructions.md` → Required sections

**Current [H9]:** "Required sections: purpose, workflow steps, output format"

**Proposed [H9]:** "Required sections: purpose, workflow steps, output format. Workflow MUST include scope enforcement (verify request falls within `scope.covers` before proceeding)."

### Change 5: Add alignment note to `pe-agents.instructions.md` (✅ done)

**Location:** Three-tier boundaries section

**Proposed addition (note/rule):**

```markdown
Three-tier boundaries MUST be consistent with YAML `boundaries:` field. Every MUST/NEVER in YAML metadata MUST appear as an Always Do or Never Do entry. Violations indicate metadata-body drift.
```

## 📐 Impact assessment

| Factor | Assessment |
|---|---|
| Breaking change? | Borderline — existing artifacts won't pass the new H14 check. But it's HIGH severity (not CRITICAL), so existing artifacts get a grace period. |
| Version bumps | pe-common: 1.6.0 → 1.7.0, pe-prompts: 1.4.1 → 1.5.0, pe-agents: minor bump |
| Downstream impact | All existing prompts/agents will need scope enforcement added to their workflows. This can be done incrementally during normal review cycles. |

## 🚀 Rollout strategy

Since this creates a new requirement that existing artifacts don't yet meet:

1. Add the rule as **HIGH** (not CRITICAL) — allows existing artifacts to function
2. Document that pe-meta review will flag non-compliant artifacts as HIGH findings
3. Existing artifacts are updated incrementally during their next review cycle (not a mass-update)
4. After 2–3 review cycles, assess whether to escalate to CRITICAL
