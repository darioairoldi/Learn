# PE-meta improvement: implementation update plan — review prompts gain mode-aware mutation capability

## 🎯 Objective (✅ done)

Update all pe-meta review/guidance-first prompt files and the option applicability matrix to:

1. Replace hard-coded `agent: plan` with `agent: agent` (mode-aware execution capability)
2. Replace "MUST stay read-only" boundary with mode-aware behavior: default is `--mode apply` (autonomous improvement for low-risk findings); `--mode plan` is opt-in for assessment-only
3. Add write tools (`replace_string_in_file`, `create_file`) to review prompts — used by default (apply mode), suppressed only when `--mode plan` is explicitly active
4. Add builder/optimizer handoff for complex improvements (parallel to how `pe-meta-scheduled-review` already works)
5. Fix the option applicability matrix: `--mode` for Review and Adherence from ❌ to ✅
6. Remove the error example that explicitly rejects `--mode apply` on review commands

- Vision v13 updated per `01.01` plan — provides the authoritative definition of Review's mutation posture
- Use cases updated per `01.02` plan — provides the post-assessment behavior specification that prompts implement

### Change 7: Add risk classification logic to prompt body

Each review prompt needs a section (or reference to shared snippet) that implements the risk classification:

```markdown
## Risk classification (when `--mode apply` is active)

For each finding, classify before acting:

1. **Non-breaking** = preserves goal, scope, and boundaries of the target artifact
   - Structural fixes (formatting, YAML, reference links, metadata) → ALWAYS non-breaking
   - Content additions that extend coverage without changing scope → USUALLY non-breaking
   - Typo fixes, consistency normalization → ALWAYS non-breaking

2. **Breaking** = changes goal, scope, or boundaries
   - Removing sections or boundaries
   - Changing artifact purpose or target audience
   - Modifying handoff chains or tool restrictions
   - Altering scope patterns

3. **Action by classification:**
   - Non-breaking + high confidence → Apply autonomously, report in summary
   - Non-breaking + medium confidence → Apply with pre-notification
   - Breaking or uncertain → Report as proposal, require human confirmation
```

**Implementation option:** Create a shared prompt snippet (`.github/prompt-snippets/pe-meta-risk-classification.md`) and reference it from all 10 prompts via `#file:` to avoid duplication. (✅ done)

## ⚠️ Risk assessment

| Risk | Mitigation |
|---|---|
| Review prompts accidentally apply changes when `--mode` isn't specified | This is now the INTENDED default behavior — `apply` is the default per vision v13. Risk classification ensures only non-breaking changes are applied autonomously. Users who want read-only output must explicitly use `--mode plan`. |
| Risk classification is too permissive (applies breaking changes) | Conservative classification: when uncertain, propose rather than apply |
| Prompt snippet duplication if risk-classification logic is inlined | Create shared snippet (`prompt-snippets/pe-meta-risk-classification.md`), reference via `#file:` |
| Handoff to builder/optimizer may not exist yet for all types | Document which handoffs are available; graceful fallback to "propose" when delegation target doesn't exist |
