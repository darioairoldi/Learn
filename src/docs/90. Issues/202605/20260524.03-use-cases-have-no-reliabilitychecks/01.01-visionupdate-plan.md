## ⚙️ Deliverables

### Change 1: Goal-trio↔catalog mapping table (✅ done)

**Location:** End of "Goal" section, before "Key definitions".

**Proposed:**

```markdown
### Goal-trio↔use-case mapping

The three goal poles (reliability, effectiveness, efficiency) are the canonical organizing axis for the use-case catalog. Every use case MUST serve at least one pole. The catalog folder structure mirrors this mapping:

| Goal pole | Use-case folder | Canonical `--dim` group |
|---|---|---|
| Cross-cutting (drives all three) | `01-freshness/` | `--dim freshness` |
| Effectiveness — guidance quality | `02-quality-gates/` | `--dim quality` |
| Effectiveness — consumer adherence | `03-consumer-correctness/` | `--dim adherence` |
| Reliability | `05-reliability/` | `--dim reliability` |
| Efficiency | `04-efficiency/` | `--dim optimize` |

Folders are numbered by historical addition order, not pole priority. A use case in any folder MAY touch dimensions from other poles, but its primary subject MUST match its folder.
```

### Change 2: Add success criterion for reliability coverage (✅ done)

**Location:** "Success criteria" section.

**Proposed:**

```markdown
- **Reliability coverage.** Every reliability anchor declared in `scope.covers` or in the rationales (loop stability, multi-pass validation, rollback safety, metadata-guarded protocol, fail-closed boundaries, boundary actionability, autonomy calibration, conflict-resolution precedence, portability boundary, mode-vs-risk decoupling) MUST be addressed by at least one use case in `05-reliability/`.
```

### Change 3: Deprecate "robustness" terminology (✅ done)

**Location:** v13 changelog block (top of document) + new entry in the conflict-resolution table.

**Proposed changelog addition:**

```markdown
> **v13 changelog (continued).** Deprecates the "robustness" `--dim` alias in favor of "adherence" (consumer-level) and "reliability" (system-level). The original alias conflated process-reliability with consumer-adherence; the rename restores the standard SE meaning of robustness as a reliability property. Back-compat: `--dim robustness` MUST resolve to `--dim adherence` with a deprecation notice for one release window.
```

**Proposed conflict-resolution row:**

```markdown
| Terminology overload (robustness alias) | "Robustness" is a reliability property in standard SE usage. The legacy `--dim robustness` alias bound to adherence dimensions is renamed to `--dim adherence`. The name "robustness" is freed for the new `05-reliability/` cluster (or kept synonymous with `--dim reliability`). |
```
