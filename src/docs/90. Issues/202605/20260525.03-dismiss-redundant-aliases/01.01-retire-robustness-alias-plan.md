## ⚠️ Risk and rollback

**Breaking risk:** Any external caller using `--dim robustness` will start getting CF-05 rejection. Migration is a one-token edit (`robustness` → `adherence` or `reliability`). CF-05 enumerates both options.

**Rollback:** Revert this commit. Since all edits land in a single coordinated change, rollback restores the deprecated-alias state cleanly. No data migration required.

**Non-rollback assets:** The lessons learned in [analysis.md](analysis.md) (especially "renames across disjoint semantic domains are not aliases" and "vision documents must not self-contradict") stand independent of this plan landing — keep them recorded even if the plan rolls back.
