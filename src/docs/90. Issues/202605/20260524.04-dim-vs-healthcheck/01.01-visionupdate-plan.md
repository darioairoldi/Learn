# Vision update plan — dismiss preset aliases

## 🎯 Goal

Remove every occurrence of `healthcheck`, `performancecheck`, and `fullcheck` as named presets from [20260523.01-vision.v13.md](../../../../../06.00-idea/self-updating-prompt-engineering/20260523.01-vision.v13.md) and replace them with the explicit canonical option set they were aliasing.

## 🔑 Canonical expansion (single source of truth)

This table governs every replacement made by this plan and by [01.02-use-cases-update-plan.md](01.02-use-cases-update-plan.md) and [01.03-pe-meta-update-plan.md](01.03-pe-meta-update-plan.md).

| Removed preset | Canonical replacement | Notes |
|---|---|---|
| `healthcheck` | `--mode plan --skip research` | Assessment-only, no external source fetch |
| `performancecheck` | `--mode apply --dim optimize --skip research,structure,consistency` | Efficiency-focused apply, delegates to `@meta-optimizer` |
| `fullcheck` | `--mode apply` | Default mode + no skip; equivalent to omitting `--mode` entirely |

**Composition rule.** When a preset already appeared alongside `--mode` / `--dim` / `--skip` flags, the explicit user flags win and the preset's contribution is merged with deduplication:

- `healthcheck --skip research` → `--mode plan --skip research` (no duplicate `research`)
- `healthcheck --dim quality --skip structure` → `--mode plan --dim quality --skip research,structure`
- `performancecheck --dim full` → `--mode apply --dim full --skip research,structure,consistency` (explicit `--dim full` overrides preset's `--dim optimize`)
- `fullcheck --scope context --mode apply` → `--mode apply --scope context` (preset is a no-op)
