# Use-cases update plan — expand preset aliases

## 🎯 Goal

Replace every `healthcheck`, `performancecheck`, and `fullcheck` token in the pe-meta use-case catalog with the explicit canonical option syntax defined in [01.01-visionupdate-plan.md](01.01-visionupdate-plan.md).

## 🔑 Canonical expansion

See [01.01-visionupdate-plan.md § Canonical expansion](01.01-visionupdate-plan.md#-canonical-expansion-single-source-of-truth). Summary for quick reference:

| Removed preset | Canonical replacement |
|---|---|
| `healthcheck` | `--mode plan --skip research` |
| `performancecheck` | `--mode apply --dim optimize --skip research,structure,consistency` |
| `fullcheck` | `--mode apply` |

Merging rule: when a preset already appeared alongside `--mode` / `--dim` / `--skip`, explicit user flags win; `--skip` stages from preset and explicit usage are merged (deduplicated).

## 🔎 Pre-execution validation

Before starting Step 1, confirm:

1. [01.01-visionupdate-plan.md](01.01-visionupdate-plan.md) has been executed and the vision file is preset-free. Use-case rewrites are stale if the vision still defines presets.
2. The canonical expansion table is the same in this plan and the vision plan.
3. The user has not requested a transitional period (e.g., "keep aliases as deprecated"). If they have, see [§ Optional transition mode](#-optional-transition-mode) instead.

If any of these are false, STOP.

### Optional transition mode

If a deprecation transition is requested (not the default scope of this plan), the rewrite still happens — but `usecase-index.json` gains a `legacy_entry_point` field per entry preserving the old string, and the catalog README adds a "Deprecated aliases" callout. Implement only if explicitly approved; otherwise skip.
