# PE-meta improvement plan: closing vision v7 gaps

## 🎯 Problem summary

Systematic comparison of the 4 pe-meta agents and 6 pe-meta prompts against vision v7 reveals **5 gaps**. The pe-meta layer provides strong coverage of the vision's Detect → Assess → Propose → Execute architecture, but the ecosystem audit path (pe-meta-validator Mode 3) is missing two checks that the vision explicitly requires, and three agents lack refinements needed for full classification and reconciliation fidelity.

**Primary concern**: When reviewing pe-gra and pe-con artifacts via ecosystem audit, instruction minimization violations and N-1 adoption gaps will go undetected — silently undermining portability (R-S8) and self-update readiness (R-P4).

### G1 (✅ Done): Add instruction minimization check to pe-meta-validator ecosystem audit

**File**: `.github/agents/00.09-pe-meta/pe-meta-validator.agent.md`
**Operation**: Update — add audit step A6 to Mode 3
**Layer**: L3 (agent)
**Classification**: Non-breaking (additive — new audit step, no existing steps changed)
**Confidence**: Deterministic — adds a new numbered step to an existing sequence

**What to add** (after A5 in Mode 3: Ecosystem Audit):

```markdown
6. **A6. Instruction Minimization** — for each instruction file in scope:
   a. Read the file and classify every rule as **testable/mechanical** (boolean pass/fail without LLM judgment) or **behavioral/strategic** (requires interpretation, tone, or style judgment)
   b. Behavioral/strategic rules in instruction files = **HIGH severity** finding (R-S8-instruction-minimization violation)
   c. Recommend moving behavioral rules to context files or agent bodies where consumers choose what to load
   d. Examples: ✅ "YAML MUST have `goal:` field" (testable). ❌ "Write in a warm, conversational tone" (behavioral)
```

**Also add** to the "✅ Always Do" boundaries section:
```markdown
- In Audit mode: check instruction files for R-S8 instruction minimization compliance (testable vs behavioral rule classification)
```

**Rollback**: Remove A6 step and the Always Do item.

## ✅ Validation criteria

After applying all 5 changes:

1. **pe-meta-validator** Mode 3 audit produces findings for:
   - Instruction files with behavioral/strategic rules (R-S8 minimization)
   - Context files and instruction files missing N-1 labels in rule-bearing sections
2. **pe-meta-designer** change specifications include:
   - Three-tier classification confidence in the Classification field
   - Tier number for blast radius ordering
3. **pe-meta-optimizer** runs metadata reconciliation after every optimization cycle
4. **No existing capabilities removed** — all changes are additive
5. **All modified agents remain within token budget** (agent budget per 01.06)
