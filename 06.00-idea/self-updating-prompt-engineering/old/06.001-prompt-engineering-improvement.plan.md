# Consolidated agents gap fix plan

> **Goal**: Close the 7 critical/high capability gaps in the consolidated agents identified during Step 8 validation, without exceeding the 200-line limit.

| Category | Count |
|---|---|

## 📊 Gap classification and fix plan

### CRITICAL gaps — fix before real-world use

| # | Gap | Agent | Fix | Lines added |
|---|---|---|---|---|
| 1 | **Phase 2.5: Impact Classification** — 3-tier breaking/non-breaking protocol missing | Researcher | Add `📖` reference to `02.05-agent-workflow-patterns.md` + brief inline Phase 2.5 section | ~8 |
| 2 | **Reversibility protocol** — no pre-change snapshot or revert-on-failure | Builder | Add reversibility bullet to ✅ Always Do section referencing the vision's Execute § Rollback | ~3 |
| 3 | **Output templates** — no `📖` references to report/handoff templates | Researcher + Builder | Add `📖` references to `output-researcher-report.template.md` and `output-builder-handoff.template.md` | ~2 each |

### HIGH gaps — fix to match granular quality

| # | Gap | Agent | Fix | Lines added |
|---|---|---|---|---|
| 4 | **Domain expertise activation** — won't load domain-specific context for cross-domain work | All 3 | Add `📖 Domain expertise activation: 02.05-agent-workflow-patterns.md` to ✅ Always Do | ~1 each |
| 5 | **Internet research validation** — no guardrail against hallucinated sources from `fetch_webpage` | Researcher | Add `📖 Internet research validation: 02.05-agent-workflow-patterns.md` to 🚫 Never Do | ~1 |
| 6 | **Phase 0.2: Input quality challenge** — won't early-reject poorly formed inputs | Researcher | Add `📖 Input quality challenge: 02.04-agent-shared-patterns.md → Phase 0.2` to ✅ Always Do | ~1 |
| 7 | **Breaking change signal table** — mentions detection but has no signal guidance | Builder | Add `📖 Breaking change signals: 02.05-agent-workflow-patterns.md` to Phase 1 | ~1 |

### MEDIUM gaps — acceptable trade-offs (NOT fixed in agents)

| # | Gap | Why acceptable |
|---|---|---|
| 8 | Hook-specific: lifecycle event matrix, security analysis | Lives in `pe-hooks.instructions.md` — loaded via dispatch table |
| 9 | Hook-specific: timeout assessment, companion script | Same — instruction file depth |
| 10 | Context-specific: layer inventory, gap/overlap analysis | Lives in `pe-context-files.instructions.md` — loaded via dispatch table |
| 11 | Context-specific: architecture challenge, topic scope | Same — instruction file depth |
| 12 | N-1 block-level classification reference | Covered by dispatch table's type-specific validation column |
| 13 | Output schema compliance | Covered by `02.05-agent-workflow-patterns.md` folder reference |
| 14 | Complexity gate in builder/validator | Already present in researcher; builder/validator don't gate on complexity |
| 15 | Single-source-of-truth validation | Covered by context file instruction rules |

**Rationale for not fixing MEDIUM gaps**: These represent the expected trade-off of consolidation per Option D — type-specific depth lives in instruction files and context files loaded dynamically via the dispatch table, not duplicated in agent bodies. The consolidated agent's job is to load the right instruction file; the instruction file provides the domain depth.
