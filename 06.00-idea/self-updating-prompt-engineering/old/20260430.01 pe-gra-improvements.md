## Priority 3 — Handoff contracts

### Fix 3.1: Add handoff data contract tables to 8 builders

**Problem**: Builders receive data from researchers and send results to validators. Without field specs, token budgets bloat and critical data gets dropped during multi-agent orchestration.

**Template to add** (adapt per builder):

```markdown
## Handoff Contract

| Direction | Fields | Max Tokens |
|---|---|---|
| Receives from researcher | gap_analysis, recommendations, constraints, affected_consumers | 2000 |
| Sends to validator | file_path, change_summary, affected_consumers, version_delta | 1500 |
```

**Effort**: Medium — requires per-builder field analysis.

**Decision**: Embed inline as compact table between boundaries and process section. Each builder gets a 3-row table (receives from researcher, sends to validator, receives back from validator) with template references and max token budgets. Required fields reference Phase 0 field tables (already in `agent-patterns`) rather than duplicating.

**Status**: ✅ DONE (2026-05-01). All 8 builders now have `## Handoff Data Contract` section with partner names, templates, and token budgets.

## Priority 5 — Structural polish (low urgency)

### Fix 5.1: N-1 labeling in Phase 0.5 decision rules (validators)

**Problem**: Phase 0.5 decision logic ("If COSMETIC: Report…") functions as rules but isn't wrapped in `**Rule:**`/`**Rationale:**` blocks per N-1 structural separation criteria.

**Scope**: All 8 validators.

**Assessment**: LOW priority — the decision logic is clear and functional. N-1 labeling adds maintenance cost without materially improving comprehension for these short decision trees.

**Decision**: DEFER unless a future review explicitly flags readability issues.

**Implementation (2026-05-01)**: Added inline `**Rationale:**` after each classification rule (COSMETIC/STRUCTURAL/VOCABULARY/BEHAVIORAL) in all 8 validators. Also labeled safety net rules in context-validator (Risk Level 2) and instruction-validator (Risk Level 1 + copilot-instructions.md special case) with `**Rule:**`/`**Rationale:**` blocks. Lightweight N-1 treatment — adds reasoning without full 3-part blocks.

**Status**: ✅ DONE.

---

### Fix 5.2: Extract shared builder boilerplate to prompt-snippet

**Problem**: Pre-save compatibility gate, reversibility checks, and post-change reconciliation sections are ~40 lines identical across all 8 builders. This is a deduplication candidate.

**Option A**: Extract to `.github/prompt-snippets/pe-builder-shared-gates.md` and reference via `#file:`.

**Option B**: Leave as-is — duplication ensures each builder is self-contained (no external dependency at runtime).

**Decision**: DEFER — Option B has operational advantages (no broken chains if snippet changes). Revisit when builders are updated for other reasons.

**Re-assessment (2026-05-01)**: Deep analysis revealed the 3 blocks are NOT identical across builders:
- **Pre-change compatibility gate**: 8 unique variants (type-specific breaking change examples, different metadata fields to check)
- **Reversibility**: 4 variants (backtick formatting, version/hash vs content-only, JSON-specific)
- **Post-change reconciliation**: Only truly identical block (5 lines × 8 = 40 lines)

Externalizing would save only ~40 lines (the reconciliation block), not the 200+ estimated. Pre-change gate CANNOT be extracted because each builder has type-specific classification logic.

**Status**: ✅ NO ACTION NEEDED. Duplication is justified — blocks are semantically customized per artifact type.

---

### Fix 5.3: Standardize researcher output template references

**Problem**: Agent-researcher and prompt-researcher reference `output-researcher-report.template.md`. The other 6 researchers embed report templates inline.

**Assessment**: Both patterns work. Inline templates are self-contained; external templates enable consistency enforcement. LOW priority — revisit during next template audit.

**Re-assessment (2026-05-01)**: Deep analysis revealed the inline templates are artifact-specific and CANNOT use the generic `output-researcher-report.template.md`:
- context-researcher: Coverage Map + Knowledge Gaps + Architecture Recommendations (5 unique tables)
- instruction-researcher: applyTo Conflicts + Layer Boundary Issues (5 unique tables)
- skill-researcher: Coverage Map + Gaps/Overlaps (3 unique tables)
- template-researcher: Missing Templates + Consumer Impact Summary (5 unique tables)
- hook-researcher: Event Coverage + Security Assessment (3 unique tables)
- snippet-researcher: Coverage Map + Issues (3 unique tables)

Only agent-researcher and prompt-researcher use the generic template. The 6 inline templates provide artifact-specific structure that the generic template lacks.

**Status**: ✅ NO ACTION NEEDED. Inline templates are intentionally specialized — standardizing would lose artifact-specific analysis structure.
