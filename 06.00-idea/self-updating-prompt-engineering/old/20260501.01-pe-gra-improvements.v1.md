# PE Slash-Command Reference Fixes

> Systematic fix of 174 broken slash-command references across 29 files in the PE system.

---

## Root cause

When pe-granular prompts were created with the `pe-gra-` prefix naming convention, cross-references in "Out of Scope" sections and context documentation were never updated to match. Similarly, when pe-meta prompts were renamed from `meta-prompt-engineering-*` to `pe-meta-*`, references in context files and tasks.json were left stale.

## Prevention

To prevent recurrence, the following checks should catch this class of error:

1. **pe-healthcheck hook** — already runs on SessionStart; could add a slash-command reference integrity check
2. **Validator agents** — Phase 0.5 VOCABULARY classification already greps for renamed terms; this would catch future naming changes
3. **Convention**: All cross-references in "Out of Scope" sections MUST use the full `name:` field value from the target prompt's YAML frontmatter

## Root cause analysis: Why pe-meta didn't catch this

### The blind spot

The PE system has **3 reference types** but only validates **2**:

| Reference type | Example | Validated? | By whom |
|---|---|---|---|
| `📖` file references | `📖 agent-patterns files` | ✅ | H12 rule + coherence-check skill Workflow 1 |
| Markdown links | `[text](path)` | ✅ | H12 rule + coherence-check skill Workflow 1 |
| **Slash-command references** | `/pe-gra-prompt-design` | ❌ | Nobody |

### Why each layer missed it

| Layer | What it checks | Why it missed slash-commands |
|---|---|---|
| **H12 rule** (critical rules matrix) | "All `📖` references and markdown links resolve" | Explicitly scoped to `📖` and markdown links — slash-commands aren't mentioned |
| **Coherence-check skill** (Workflow 1) | Scans for `📖` + markdown link patterns | Pattern list doesn't include `/[a-z-]+` slash-command pattern |
| **Prompt-validator** (Phases 1–6) | YAML fields, tool alignment, boundaries, production readiness | "Out of Scope" sections treated as free-form prose — body text names never parsed |
| **pe-meta-review** (strategic) | Vision alignment, L1.5 categories, quality bar, N-1 | Category ref check only applies to `📖` context file references |
| **Phase 0.5** (change impact) | VOCABULARY changes grep for renamed terms | Only triggered post-change — doesn't audit pre-existing references |

### Why it persisted across multiple review cycles

We ran pe-meta on all 24 pe-granular agents — it correctly flagged Level 2 `📖` references, missing handoff contracts, and boundary gaps. But slash-command references live in **prompt files** (not agent files), and the prompt-validator's checklist has no "body text reference resolution" check. The broken references were in plain sight in every "Out of Scope" section, but no validation rule asks "does this `/name` match an existing prompt's `name:` field?"
