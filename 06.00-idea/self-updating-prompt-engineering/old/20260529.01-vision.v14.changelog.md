# Self-updating PE vision — changelog

> Most recent changes first. The vision document itself ([20260529.01-vision.v14.md](20260529.01-vision.v14.md)) carries only a short summary of recent changes; the full history lives here.

## v15 — 2026-05-30

**Domain-coherent batching as a first-class invocation contract (R-P10).** When R-P8 (default-full) + R-P9 (minimal option surface) combine to resolve a parameter-less invocation into a multi-domain batch, the orchestrator MUST surface the domain footprint and propose a per-domain split BEFORE Phase 1 runs. Hard-gated split proposal in `--mode apply`; advisory proposal in `--mode plan`. Bypassable only through explicit `bundle=accept` consent on the invocation line. Reference: [01-vision-update-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/03-pe-meta-update-applied-to-all-pe-contexts/01-vision-update-plan.md).

**New Phase 0b — Domain coherence check (never skippable).** Inserted deterministically between Phase 0a (conversational pre-parser) and Phase 1 (research). Computes the domain footprint of the resolved scope via the metadata-first 3-tier resolution algorithm. `--skip domain-coherence` is REJECTED with CF-05; the gate can only be retired by a single-domain footprint, explicit `bundle=accept`, accepting a proposed per-domain split, or the `bundle=cross-domain-deps` carve-out.

**Phase 0a CF-05 artifact-type/path consistency check.** Per-artifact prompts (`pe-meta-context-*`, `pe-meta-prompt-*`, `pe-meta-instruction-*`, `pe-meta-agent-*`, `pe-meta-template-*`, `pe-meta-skill-*`, `pe-meta-hook-*`, `pe-meta-snippet-*`) reject invocations whose positional path is outside the artifact-type root encoded in the prompt name. Orchestrator-level prompts (`pe-meta-update`, `pe-meta-review`, `pe-meta-create-update`, `pe-meta-design`, `pe-meta-scheduled-review`, `pe-meta-release-monitor`, `pe-meta-adherence`) are artifact-type-agnostic and skip this check.

**Metadata-first 3-tier domain resolution algorithm.** Domain is a content property declared in artifact metadata, NOT a path property. Tier 1: declared `domain:` frontmatter (authoritative). Tier 2: optional per-repo `pe-domain-map.yaml` path-slug heuristic (flagged in Phase 8 report). Tier 3: `unknown` (per-file warning). Replaces v14's implicit path-walk model.

**Seed-footprint vs dependency-footprint distinction with `bundle=cross-domain-deps` carve-out.** Phase 0b computes domain footprints separately for the seed scope and for the `--deps direct`/`--deps full` closure. When seed = 1 domain and deps add ≥ 1 additional domain, the bundle disposition is `cross-domain-deps` (NOT a split) — the union runs as ONE review with per-dependency-domain specialized analysis lenses in Phase 2–4 and per-dependency-domain report sectioning in Phase 8. Splitting would produce incomplete reviews because consumer artifacts need ALL declared dependencies to be reviewable. R-P10 (which governs seed-multi-domain) is unchanged.

**Per-invocation-type scope-extraction matrix (orthogonal to domain resolution).** Five invocation shapes (artifact-type token, path-set single, path-set multi, positional `<file-path>`, no-scope default-all) all feed the same Phase 0b downstream gate. The `scope-source=` marker records which shape was used so audit trails distinguish positional invocations from token-based ones.

**Eight canonical artifact-type roots formalized.** `.copilot/context/`, `.github/prompts/`, `.github/instructions/`, `.github/agents/`, `.github/templates/`, `.github/skills/`, `.github/hooks/`, `.github/prompt-snippets/`. Used for scope expansion and CF-05 only — they are NOT a semantic-domain partitioning.

**Optional `pe-domain-map.yaml` for cross-repo portability.** Per-repo path-slug heuristic that other repositories MAY ship at root to back-fill Tier 2 resolution for legacy/unannotated files. Schema: `mappings: [- path-prefix: …, domain: …]`. The vision specifies WHAT to read (`domain:` field) and HOW to fall back; it does NOT mandate any folder-naming convention. Repos that declare `domain:` in every artifact's frontmatter never need this file.

**Success criterion #12 extended with new markers.** The first-line `Resolved invocation:` log appends `scope-source=` (`token` | `path-set` | `positional` | `default-all`) and `bundle=` (`single-domain` | `multi-domain-gated` | `accepted-bundle` | `split-N` | `cross-domain-deps`). The Phase 8 footprint table additionally emits a per-file `domain-source=` column (`declared-metadata` | `path-heuristic` | `unknown`). Linter rejects entries missing any of these markers.

**v14 documentation-state corruption repaired (lines 1288–1303).** A previously crushed single-line dump in § Pipeline phases (containing literal `\n`, `\u2014`, `\u2192`, `\u2286` escape sequences embedded as text — skip rules 4–6, Reference paragraph, entire § Conversational pre-parser section, entire § Per-artifact prompt invocation matrix section, and the § Mode elimination heading) is restored to properly-formatted multi-line markdown. Pipeline-phase table em-dashes and skip rule #2 em-dash similarly decoded.

**Companion plans.** This v15 cycle was executed via three coordinated plans: [01-vision-update-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/03-pe-meta-update-applied-to-all-pe-contexts/01-vision-update-plan.md) (the vision changes themselves), [02-context-updates-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/03-pe-meta-update-applied-to-all-pe-contexts/02-context-updates-plan.md) (downstream context-file updates), and [03-prompts-updates-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/03-pe-meta-update-applied-to-all-pe-contexts/03-prompts-updates-plan.md) (downstream `/pe-meta-*` prompt updates).

## v14 — 2026-05-29

**Default-full invocation contract generalized to all manual PE-meta commands.** A parameter-less *manual* invocation of any `/pe-meta-*` command (not only `/pe-meta-update`) is contracted as a full review across every in-scope artifact, every monitored source, deep per-artifact tier, organizational-pass-first. Trigger-fired invocations resolve to incremental. Parameters are subtractive — the system never opts into a narrower scope on the user's behalf.

**Canonical option surface collapsed to seven minimal-and-consistent parameters.** `--mode`, `--scope`, `--source`, `--dim`, `--start`/`--end`, `--deps`, `--skip`. Every additional flag risks overlap, inconsistency, or silent re-interpretation downstream (R-P9).

**Value-shape `--scope` parser.** `--scope` accepts either an artifact-type token (`context | instructions | agents | prompts | skills | hooks | snippets | templates | all`) OR a comma-separated set of target folders/files. Absorbs the v13 `--area` and `--artifact` flags.

**Phase 0a — conversational pre-parser.** Resolves free-form input to canonical options before strict parsing. Phases 1–8 only ever consume canonical options. Surface gaps via a single clarification question — never silently invent.

**Breadth is a resolved attribute, not a flag.** `full` / `incremental` / `bounded-delta` is derived from `--start`/`--end` plus caller-type (interactive vs trigger-fired) and logged on the first line of every run report.

**Research contract.** Research-phase output is either a full **snapshot** (manual full) or a bounded **change digest** (trigger-fired incremental / bounded-delta). Both feed Tier-2 relevance screening.

**Two new rationales.** R-P8-default-full-investigation, R-P9-minimal-consistent-option-surface.

**New subsections under § Command families and option model:**

- Conversational pre-parser (Phase 0a)
- Per-artifact prompt invocation matrix (`(--scope artifact-type, --dim)` → `pe-meta-{type}-{review | create-update | design}`)
- Pipeline phases and `--skip` mapping (explicit 8-phase pipeline with per-phase `--skip` semantics)
- Deprecated-flags migration table covering every retired v13 flag (`--breadth`, `--area`, `--artifact`, `--consumer`, `--subject`, `--concern`, `--incremental`, `--catch-up`, `--adherence`, `--mode-review`)

**New Phase 1.5 (organizational pass).** Gated on `breadth=full` AND `--scope` broader than a single file.

**Success criterion #12.** Resolved invocation is logged on every run as the first line of the report — across every `/pe-meta-*` command, regardless of caller type.

**Companion plans.** See [02-pe-meta-update-not-processing-full-context-review/01-vision-update-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/02-pe-meta-update-not-processing-full-context-review/01-vision-update-plan.md).
