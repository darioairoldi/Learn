# PE-gra improvement plan: closing vision v7 gaps

## 🎯 Problem summary

Systematic review of 24 pe-gra agents and 17 pe-gra prompts against vision v7 reveals **8 gap categories**. The pe-gra layer has strong operational quality — all agents have goals, rationales, context dependencies, test scenarios, and response management. But it falls short of the PE quality bar (exemplary level from 05.06) in metadata completeness, reference patterns, and several prompts lack operational patterns that design/review prompts already have.

**Two blocking issues**: merge conflict markers in 2 prompt files (`context-information-create-update`, `instruction-file-create-update`).

### G2 (✅ Done): Add structured `scope:` YAML to all 24 pe-gra agents

**Files**: All `.github/agents/00.02-pe-granular/*.agent.md` (24 files)
**Operation**: Update — add `scope:` field to YAML frontmatter
**Classification**: Non-breaking (Deterministic-structural — additive metadata)

**Pattern** — extract from existing `description:` and `goal:` fields:

```yaml
scope:
  covers:
    - "[primary responsibility from description]"
    - "[secondary responsibility from description]"
  excludes:
    - "[what the agent explicitly doesn't do — from body boundaries]"
```

**Batch approach**: Process all 24 agents using the same extraction pattern. Group by role (8 builders, 8 researchers, 8 validators) for consistency.

**Rollback**: Remove `scope:` fields from YAML.

---

### G3 (✅ Done): Promote `boundaries:` to YAML for all 24 pe-gra agents

**Files**: All `.github/agents/00.02-pe-granular/*.agent.md` (24 files)
**Operation**: Update — add top-level `boundaries:` to YAML frontmatter
**Classification**: Non-breaking (Deterministic-structural — additive metadata)

**Pattern** — extract 3–5 key boundary items from the body's Always Do / Never Do sections:

```yaml
boundaries:
  - "MUST NOT modify files (plan mode)" # or "MUST validate after every change"
  - "MUST load dispatch table / instruction file first"
  - "MUST rank findings by severity"
```

**Note**: Body boundaries remain — the YAML `boundaries:` field is a machine-readable summary for pre-change guards, not a replacement for the detailed body section.

**Rollback**: Remove `boundaries:` fields from YAML.

### G8 (✅ Done): Add R-S8 instruction minimization check to pe-gra-instruction-validator

**File**: `.github/agents/00.02-pe-granular/pe-gra-instruction-validator.agent.md`
**Operation**: Update — add minimization validation step
**Classification**: Non-breaking (additive check)

**What to add** to the validation process:

```markdown
### Instruction Minimization Check (R-S8)

For every rule in the instruction file:
1. Classify as **testable/mechanical** (boolean pass/fail without LLM judgment) or **behavioral/strategic** (requires interpretation)
2. Behavioral/strategic rules = **HIGH severity** finding
3. Recommend: move behavioral rules to context files or agent bodies
4. Examples: ✅ "YAML MUST have `goal:` field" (testable). ❌ "Write in a warm, conversational tone" (behavioral)
```

**Also add** to Always Do boundaries:
```markdown
- Classify every rule as testable/mechanical vs behavioral/strategic (R-S8 instruction minimization)
```

**Why this is separate from G1-G2 in the pe-meta plan**: The pe-meta-validator checks minimization during ecosystem audits (Mode 3). This gap is about the pe-gra-instruction-validator checking minimization during per-file validation — the lower layer that runs first.

**Rollback**: Remove the minimization check step and boundary item.
