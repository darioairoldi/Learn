## v15.3 ŌĆö 2026-06-04

**v15.3.0 value-shape `--start`/`--end` + Processing-state model + two P1 principles.** Makes incremental re-processing explicit and re-baselineable without adding a parameter. Companion plan: [02-pe-meta-update-plan.md](../../src/docs/90.%20Issues/202606/20260602.01-pe-meta-update/02-pe-meta-update-plan.md).

- **`--start` / `--end` extended to a value-shape parameter (date OR source-version).** A bound may now be an ISO-8601 date / `now` / relative offset OR a source-version token (e.g. `1.099`) resolved to a publish timestamp via the scoped source's `version_scheme`. Phase 0a performs the versionŌåÆtimestamp resolution; phases 1ŌĆō8 consume only a resolved time window. Mirrors the existing two-shape `--scope` parser. **No new parameter** ŌĆö the canonical surface stays seven, so `minimal-canonical-surface` (P0) is preserved and breadth remains a derived attribute. Two guard rails: a version-shaped bound REQUIRES a singleton `--source`; a source with `version_scheme: none` rejects version tokens. Updated: ┬¦ Option taxonomy parameter row + new value-shape parser paragraph, "What full means operationally" table, Default-full semantics by parameter row, Option applicability key-decision bullet, and the ┬¦ Assess manual/trigger/bounded-delta breadth list and research-phase contract table.

- **New `#### Processing-state model` subsection (under ┬¦ Assess).** Defines the two-axis state model: an **input axis** (per-source ledger `<state.path>/triggers/<source-id>.json` with `source_id`, `version_scheme`, `last_seen_version`, `last_seen_timestamp`, `last_digest_hash`) and a **processing axis** (per-artifact, per-dimension coverage with `source_versions`, `depth`, `status`, `last_run`). Specifies the **processing unit (PU = artifact ├Ś applicable-dimension)** staleness rule and three watermark origins (full = beginning of time, coverage ignored; incremental = recorded `source_versions`, coverage trusted; bounded-delta = explicit `--start`, coverage OVERRIDDEN inside the window). The bounded-delta override is the **re-baseline / distrust-recovery path** ŌĆö re-run a window whose prior processing is no longer trusted even though it was already covered.

- **Two new P1 principles.** `processing-state-is-multidimensional` (*relies on:* `metadata-driven`, `deterministic-processing`) ŌĆö state is recorded on two independent axes, never collapsed to a per-source scalar. `coverage-completeness-guarantee` (*relies on:* `validation-caching`, `deterministic-processing`) ŌĆö a never-covered PU is always in the work set; a PU is skipped only when provably current; a narrowed pass never leaves a permanent coverage gap. Matching body subsections added under ┬¦ Design principles after *Active dimensions follow assessment capability*. P0 count unchanged (5, within 3ŌĆō7); per `vision-frontmatter.instructions.md` a P1 addition requires a changelog entry (this one) ŌĆö the version bump to 15.3.0 also reflects the new behavioral contract.

- **New P2 `scope.covers` item `value-shape-start-end-parser`.** Mirrors the existing `value-shape-scope-parser` entry.

Hard-gated split proposal in `--mode apply`; advisory proposal in `--mode plan`. Bypassable only through explicit `bundle=accept` consent on the invocation line. Reference: [01-vision-update-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/03-pe-meta-update-applied-to-all-pe-contexts/01-vision-update-plan.md).

**New Phase 0b ŌĆö Domain coherence check (never skippable).** Inserted deterministically between Phase 0a (conversational pre-parser) and Phase 1 (research). Computes the domain footprint of the resolved scope via the metadata-first 3-tier resolution algorithm. `--skip domain-coherence` is REJECTED with CF-05; the gate can only be retired by a single-domain footprint, explicit `bundle=accept`, accepting a proposed per-domain split, or the `bundle=cross-domain-deps` carve-out.

**Phase 0a CF-05 artifact-type/path consistency check.** Per-artifact prompts (`pe-meta-context-*`, `pe-meta-prompt-*`, `pe-meta-instruction-*`, `pe-meta-agent-*`, `pe-meta-template-*`, `pe-meta-skill-*`, `pe-meta-hook-*`, `pe-meta-snippet-*`) reject invocations whose positional path is outside the artifact-type root encoded in the prompt name. **Metadata-first 3-tier domain resolution algorithm.** Domain is a content property declared in artifact metadata, NOT a path property. Tier 1: declared `domain:` frontmatter (authoritative). Tier 2: optional per-repo `pe-domain-map.yaml` path-slug heuristic (flagged in Phase 8 report). Tier 3: `unknown` (per-file warning). Replaces v14's implicit path-walk model.

**Seed-footprint vs dependency-footprint distinction with `bundle=cross-domain-deps` carve-out.** Phase 0b computes domain footprints separately for the seed scope and for the `--deps direct`/`--deps full` closure. When seed = 1 domain and deps add Ōēź 1 additional domain, the bundle disposition is `cross-domain-deps` (NOT a split) ŌĆö the union runs as ONE review with per-dependency-domain specialized analysis lenses in Phase 2ŌĆō4 and per-dependency-domain report sectioning in Phase 8. Splitting would produce incomplete reviews because consumer artifacts need ALL declared dependencies to be reviewable. R-P10 (which governs seed-multi-domain) is unchanged.

**Per-invocation-type scope-extraction matrix (orthogonal to domain resolution).** Five invocation shapes (artifact-type token, path-set single, path-set multi, positional `<file-path>`, no-scope default-all) all feed the same Phase 0b downstream gate. The `scope-source=` marker records which shape was used so audit trails distinguish positional invocations from token-based ones.

**Eight canonical artifact-type roots formalized.** `.copilot/context/`, `.github/prompts/`, `.github/instructions/`, `.github/agents/`, `.github/templates/`, `.github/skills/`, `.github/hooks/`, `.github/prompt-snippets/`. Used for scope expansion and CF-05 only ŌĆö they are NOT a semantic-domain partitioning.

**Optional `pe-domain-map.yaml` for cross-repo portability.** Per-repo path-slug heuristic that other repositories MAY ship at root to back-fill Tier 2 resolution for legacy/unannotated files. Schema: `mappings: [- path-prefix: ŌĆ”, domain: ŌĆ”]`. The vision specifies WHAT to read (`domain:` field) and HOW to fall back; it does NOT mandate any folder-naming convention. Repos that declare `domain:` in every artifact's frontmatter never need this file.

## v14 ŌĆö 2026-05-29

**Default-full invocation contract generalized to all manual PE-meta commands.** A parameter-less *manual* invocation of any `/pe-meta-*` command (not only `/pe-meta-update`) is contracted as a full review across every in-scope artifact, every monitored source, deep per-artifact tier, organizational-pass-first. Trigger-fired invocations resolve to incremental. Parameters are subtractive ŌĆö the system never opts into a narrower scope on the user's behalf.

**Canonical option surface collapsed to seven minimal-and-consistent parameters.** `--mode`, `--scope`, `--source`, `--dim`, `--start`/`--end`, `--deps`, `--skip`. Every additional flag risks overlap, inconsistency, or silent re-interpretation downstream (R-P9).

**Value-shape `--scope` parser.** `--scope` accepts either an artifact-type token (`context | instructions | agents | prompts | skills | hooks | snippets | templates | all`) OR a comma-separated set of target folders/files. Absorbs the v13 `--area` and `--artifact` flags.

**Phase 0a ŌĆö conversational pre-parser.** Resolves free-form input to canonical options before strict parsing. Phases 1ŌĆō8 only ever consume canonical options. Surface gaps via a single clarification question ŌĆö never silently invent.

**Breadth is a resolved attribute, not a flag.** `full` / `incremental` / `bounded-delta` is derived from `--start`/`--end` plus caller-type (interactive vs trigger-fired) and logged on the first line of every run report.

**Research contract.** Research-phase output is either a full **snapshot** (manual full) or a bounded **change digest** (trigger-fired incremental / bounded-delta). Both feed Tier-2 relevance screening.

**Two new rationales.** R-P8-default-full-investigation, R-P9-minimal-consistent-option-surface.

**New subsections under ┬¦ Command families and option model:**

- Conversational pre-parser (Phase 0a)
- Per-artifact prompt invocation matrix (`(--scope artifact-type, --dim)` ŌåÆ `pe-meta-{type}-{review | create-update | design}`)
- Pipeline phases and `--skip` mapping (explicit 8-phase pipeline with per-phase `--skip` semantics)
- Deprecated-flags migration table covering every retired v13 flag (`--breadth`, `--area`, `--artifact`, `--consumer`, `--subject`, `--concern`, `--incremental`, `--catch-up`, `--adherence`, `--mode-review`)

**New Phase 1.5 (organizational pass).** Gated on `breadth=full` AND `--scope` broader than a single file.

**Success criterion #12.** Resolved invocation is logged on every run as the first line of the report ŌĆö across every `/pe-meta-*` command, regardless of caller type.

**Companion plans.** See [02-pe-meta-update-not-processing-full-context-review/01-vision-update-plan.md](../../src/docs/90.%20Issues/202605/20260525.03-staleness-review/02-pe-meta-update-not-processing-full-context-review/01-vision-update-plan.md).

## Historical: v13 ŌåÆ v14 deprecated flag map

> **Provenance.** Migrated verbatim from the v15 vision body (`## ­¤ō” Migration notes (v13 ŌåÆ v14)`) when v15.1.0 retired that section. Retained here as the authoritative reference for downstream parsers that emit CF-05 rejection messages for retired v13 flags. The uniform rejection template is `<flag> retired in v14; use <v14-replacement> ŌĆö see vision v15 changelog ┬¦ Historical: v13 ŌåÆ v14 deprecated flag map`.

| Retired v13 surface | v14 destination |
|---|---|
| `--breadth full\|incremental\|catch-up` | Derived attribute; logged on first line of every run report |
| `catch-up` value (any flag) | `--start <older-than-default>` on any caller; resolves to `bounded-delta` |
| `--since <date>` | `--start <date>` |
| `--between <a>..<b>` | `--start <a> --end <b>` |
| `--area <token>` | `--scope <artifact-type-token>` |
| `--artifact <path>` | `--scope <path>[,<path>...]` |
| `--consumer <path>` | `--scope <agent-or-prompt-file> --deps full` |
| `--subject <kw>` | Resolved by Phase 0a ŌåÆ comma-separated `--scope` enumeration |
| `--concern <kw>` | Resolved by Phase 0a ŌåÆ `--scope` (+ `--dim` when keyword maps to a dimension) |
| `--mode-review individual\|dep-aware\|guidance-first` | Auto-derived from `--scope` artifact-type via per-artifact prompt invocation matrix; guidance-first leg covered by `--dim adherence` |
| `--incremental` | **Single migration-window alias** ŌĆö accepted ONLY for trigger-fired callers (resolves to derived `breadth=incremental`); REJECTED for manual callers (would violate `default-full-investigation`) |
