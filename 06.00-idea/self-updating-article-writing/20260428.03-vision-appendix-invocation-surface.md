# Article maintenance — canonical invocation surface

> Appendix to [20260428.01-vision.v1.md](20260428.01-vision.v1.md). It specifies the invocation surface the `predictable-invocation-surface` principle requires. This is a contract, not an implementation — the orchestrator that executes it is parked (see the companion plan's park lot).

## Why a small surface

A wide invocation surface (many flags, per-collection defaults, preset aliases) forces the operator to remember which option applies where, and a parameter-less request with no contract gets silently narrowed to the cheapest pass. The surface answers with **eight canonical parameters**, a **default-full** contract, a **conversational pre-parser** that resolves free-form intent to those parameters, and a **resolved-scope echo** on every run.

## The eight parameters

| Parameter | Selects | Values (default in **bold**) |
|---|---|---|
| `--mode` | preview vs apply | plan (write a plan, stop) · **apply** (write a plan, then execute it) |
| `--scope` | which articles | a single article · a series · a folder · a subject · a tag set · **all** |
| `--source` | which source feed to research | a source/technology feed id · **all monitored sources** |
| `--dim` | which quality dimensions | verified · reliable · readable · understandable · up-to-date · the collection dimensions · **all applicable** |
| `--deps` | how far to traverse | none · direct · full — follow cross-references, collection links, and cited sources · **the command's natural depth** |
| `--start` / `--end` | research window | a date (`YYYY-MM-DD`, `now`, `-90d`) OR a source-version token resolved to a timestamp · **unbounded for a full run; the trigger delta for a trigger-fired run** |
| `--skip` | drop named pipeline phases | one or more phase names (see *The maintenance pipeline*) · **skip nothing** |
| `--plan-file` | where the run's plan lands | a workspace-relative path · **an auto-named path in the run folder** |

`--scope` and `--deps` compose orthogonally (target set ∪ dependency traversal); each included article is assessed with the dimensions applicable to it. `--mode plan` and `--mode apply` both write a plan file — they differ only in whether it is then executed (`plan = apply − execute`; `apply = plan + execute`). `--plan-file` sets the plan's location only; it never decides whether the plan is regenerated or trusted (see the vision's *Plan execution modes*).

## Default-full and resolved-scope echo

- A **manual, parameter-less** invocation is a **full review**: every in-scope article, every applicable dimension, every monitored source. Parameters only *narrow* the full default — the system never opts into a narrower scope on the operator's behalf.
- A **trigger-fired** invocation (freshness drop, broken link, source change, scheduled review) is **incremental**: the trigger is the delta signal.
- Every run echoes its **resolved scope** as the first line of its report — `articles × dimensions × window` — so what ran can never silently diverge from what was asked.

## Collection scope

`--scope` accepts a collection (series, folder, subject, or tag). A collection run applies the collection-structural gate (orphan, scope overlap, navigation chain) before per-article work, per the *Series before articles* (collection) principle.

## The maintenance pipeline and `--skip`

A maintenance run is an ordered set of named phases. `--skip` drops one or more of them; the never-skippable phases are marked.

| Phase | Does | Skippable? |
|---|---|---|
| inventory | enumerate in-scope articles, build the dependency/collection graph, hash inputs | never |
| pre-parser | resolve free-form intent to the canonical surface, echo the resolved scope | never |
| domain-coherence | compute the per-article domain footprint; propose a per-domain split for multi-domain sweeps | never |
| research | gather source/version deltas for the `--source` window | `--skip research` — uses the cached source ledger; fails closed if none is fresh |
| collection pass | run the collection-structural gate (orphan, scope overlap, navigation chain) | `--skip` allowed for single-article runs |
| structural tier (T0) | deterministic checks (links, freshness, structure, metadata, references) | `--skip` allowed |
| readability/understandability tier (T1) | LLM-light prose checks (readability, grammar, standards) | `--skip` allowed |
| factual/up-to-date tier (T2) | LLM-deep checks (factual accuracy, completeness, logical flow) | `--skip` allowed |
| approval | confirmation gate per the reader-risk autonomy gradient | never skippable for content-altering changes |
| apply | apply approved changes; checkpoint each for rollback | equivalent to `--mode plan` when skipped |
| regression | post-apply re-validation | forbidden to skip when apply ran |
| report | emit the resolved-scope echo and findings | never |

A skipped phase drops the dimensions that depend on its evidence — those dimensions report **not assessed**, never a low-confidence pass (the vision's *active dimensions follow evidence* rule). For example, `--skip research` still runs the structural and readability tiers but reports factual accuracy as *not assessed*.

## Conversational pre-parser

Free-form operator intent — for example *"review the Azure articles touched since 1.100 for factual drift"* — is normalized into the canonical surface before execution: `--scope` = the Azure article set, `--start` = the 1.100 source-version boundary, `--dim` = the factual/up-to-date dimensions. The pre-parser **echoes the resolved invocation** before any work begins, may ask a single clarification when intent is ambiguous, defaults to full when it cannot narrow confidently, and **never silently invents scope**. Phases downstream of the pre-parser only ever consume the resolved canonical surface.

## What this appendix is (and isn't)

This surface adds no hidden flags: the target collection and the time window are *resolved* attributes, and free-form intent resolves to the eight parameters through the pre-parser. The appendix defines the **contract** only — the dimension vocabulary and the freshness/SLA formula come from the vision, and building the orchestrator that executes this surface, the named pipeline, and the pre-parser is out of scope here and tracked separately.
