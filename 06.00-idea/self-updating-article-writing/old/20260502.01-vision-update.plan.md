## 🎯 Problem this solves

The `## 🔄 Options analysis` section occupies ~130 lines evaluating three architectural approaches. This is useful decision context but is not normative — it doesn't define standards, rules, or principles. It breaks the flow between Goal (what we want) and Rationale (why these strategies):

**Current flow:** Problem → Goal → *Options analysis (evaluation)* → Rationale → Vision → Principles

**Target flow:** Problem → Goal → Rationale → Vision → Principles

The options section also mixes two concerns:
- **Option evaluation** (batch vs event-driven vs rewrite) — belongs in an appendix
- **System architecture** (diagram, three-systems-roles table, cross-system triggers) — describes the chosen architecture and belongs in The vision
